## ターゲットユーザー
社内申請手続きを行う一般社員（申請ルールや申請方法に不慣れなユーザーを含む）

## 要求事項

- RQ-01: 社員が申請内容を自然言語で入力すると、必要な申請種別・申請書・申請先を提示する
- RQ-02: 申請ルールを参照して申請種別を自動判断する
- RQ-03: 不足情報を対話で確認しながら申請書を自動作成する
- RQ-04: 申請ミスや差し戻しを減らす
- RQ-05: 申請ルールや申請方法が分からない利用者でも正しく申請できる

## 業務要件

| ID    | 業務要件 | 関連要求  | AI関与 | 優先度 |
| ----- | -------- | --------- | ------ | ------ |
| BR-01 | 社員が入力した申請内容から申請種別（交通費精算申請・経費精算申請）を自動判断する | RQ-01, RQ-02 | 実行 | High |
| BR-02 | 判断した申請種別に対応する申請書名と申請先を社員に提示する | RQ-01 | 実行 | High |
| BR-03 | 申請書作成に必要な情報が不足している場合、対話で確認・収集する | RQ-03 | 実行 | High |
| BR-04 | 収集した情報をもとに申請書（Excel等）を自動作成する | RQ-03, RQ-04 | 実行 | High |
| BR-05 | 作成した申請書の内容を申請ルールと照合し、ミスを検出・修正案を提示する | RQ-04 | 提案 | High |
| BR-06 | 申請種別が判断できない場合、選択肢を提示してユーザーに確認する | RQ-01, RQ-05 | 提案 | High |
| BR-07 | 申請書生成前にユーザーが内容を確認・修正できる承認ステップを設ける | RQ-04, RQ-05 | 提案 | High |

## 業務種別ごとの業務ルール

### 申請種別判断の業務ルール

| ID | 業務ルール | 詳細 | 優先度 |
| -- | ---------- | ---- | ------ |
| JD-01 | 交通費精算申請の判断 | 申請内容に「電車」「バス」「タクシー」「飛行機」「交通費」「出張」「移動」「乗車」「運賃」等のキーワードが含まれ、業務に関連する交通費の精算を目的とする場合 | High |
| JD-02 | 経費精算申請の判断 | 申請内容に「経費」「領収書」「事務用品」「宿泊」「資格」「購入」「精算」等のキーワードが含まれ、業務に関連する経費の精算を目的とする場合 | High |
| JD-03 | 判断不能時の対応 | 上記いずれにも明確に該当しない場合、または複数の申請種別に該当する可能性がある場合は、選択肢（交通費精算申請・経費精算申請）を提示してユーザーに確認する | High |

### 申請書作成の業務ルール

| ID | 業務ルール | 詳細 | 優先度 |
| -- | ---------- | ---- | ------ |
| AP-01 | 必須項目の収集 | 申請種別ごとに定められた必須項目が揃うまで対話で確認する | High |
| AP-02 | 業務目的の必須記載 | すべての申請種別において業務目的の記載を必須とする | High |
| AP-03 | 申請期限チェック | 経費・交通費精算申請は経費発生日から3ヶ月以内であることを確認する | High |
| AP-04 | 上長承認要否の通知 | 申請金額が閾値を超える場合、上長承認が必要であることを通知する | High |
| AP-05 | 申請書生成前の確認 | 申請書生成前にユーザーが内容を確認し、OK/修正/キャンセルを選択できる | High |
| AP-06 | 申請書の検証 | 生成した申請書の内容を申請ルールと照合し、ミスを検出して修正案を提示する | High |
