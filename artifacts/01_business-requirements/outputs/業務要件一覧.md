## ターゲットユーザー
社内の一般社員（申請ルールや申請方法に詳しくない場合がある）

## 要求事項
- RQ-01: 社員が申請内容を入力すると、必要な申請種別・申請書・申請先を提示する
- RQ-02: 社内申請ルールを参照して申請種別を自動判断する
- RQ-03: 不足情報を対話で確認しながら申請書を自動作成する
- RQ-04: 申請ミスや差し戻しを減らす
- RQ-05: 申請ルールや申請方法が分からない利用者でも正しく申請できる

| ----- | -------- | --------- | ------ | ------ |
| BR-01 | 申請内容の入力を受け付ける | RQ-01, RQ-05 | 実行 | High |
| BR-02 | 申請内容から交通費精算申請・経費精算申請のいずれかを判断する | RQ-01, RQ-02 | 実行 | High |
| BR-03 | 使用すべき申請書と申請先を提示する | RQ-01 | 提案 | High |

| BR-07 | 申請ルール・方法が不明な利用者を支援する | RQ-05 | 提案 | High |

## 業務種別ごとの業務ルール

### 申請種別判断の業務ルール

| ID | 業務ルール | 詳細 | 優先度 |
| -- | ---------- | ---- | ------ |
| APL-01 | 申請種別の特定 | 社員の申請内容（目的・状況）から、交通費精算申請・経費精算申請のいずれかを判断する | High |
| APL-02 | 申請種別判断不能時の確認 | 申請内容から交通費精算申請・経費精算申請のいずれかを判断できない場合は、ユーザーに選択肢（「交通費精算申請」「経費精算申請」）を提示して確認する | High |
| APL-03 | 申請先の特定 | 申請種別に応じた申請先（部署・担当者・システム等）を提示する | High |
| APL-04 | 不足情報の確認 | 申請書作成に必要な情報が不足している場合、対話で確認する | High |
| APL-05 | 申請書の自動作成 | 確認済み情報をもとに申請書を自動作成する | High |
| APL-06 | 申請ルール参照 | 申請判断は社内申請ルール（ナレッジ）を参照して行う | High |

### 交通費精算申請の業務ルール

| ID | 業務ルール | 詳細 | 優先度 |
| -- | ---------- | ---- | ------ |
| TRV-01 | 移動情報の収集単位 | 移動情報（移動日、出発地、目的地、交通手段、費用、業務目的）を一区間ずつ収集する | High |
| TRV-02 | 対応交通手段 | 電車、バス、タクシー、飛行機の4種類を対象とする | High |
| TRV-03 | 交通費の自動計算 | 運賃データから交通費を自動計算する（電車は経路テーブル検索、バス・タクシー・飛行機は固定運賃） | High |
| TRV-04 | 申請期限 | 申請は経費発生日から3ヶ月以内であること | High |
| TRV-05 | 上長承認要件 | 交通費が10,000円を超える場合は事前に上長の承認が必要 | High |
| TRV-06 | 駅名の正規化 | 駅名を正規化する（例：「渋谷駅」→「渋谷」） | Medium |
| TRV-07 | 業務目的の記載 | すべての申請に業務目的の記載が必要 | High |

### 経費精算申請の業務ルール

| ID | 業務ルール | 詳細 | 優先度 |
| -- | ---------- | ---- | ------ |
| EXP-01 | 領収書からの情報抽出 | 領収書画像から経費情報（購入日、店舗名、品目、金額）を品目ごとに自動抽出する | High |
| EXP-02 | 経費区分の自動判断 | 品目から経費区分を自動判断する（事務用品費、宿泊費、資格精算費、その他経費の4区分） | High |
| EXP-03 | 申請期限 | 申請は経費発生日から3ヶ月以内であること | High |
| EXP-04 | 上長承認要件 | 経費が5,000円を超える場合は事前に上長の承認が必要 | High |
| EXP-05 | 業務目的の記載 | すべての申請に業務目的の記載が必要 | High |

## 申請対象

### 交通費精算申請の対象交通手段
- 電車
- バス
- タクシー
- 飛行機

### 経費精算申請の対象経費区分
- 事務用品費（書籍、文房具、オフィス用品など）
- 宿泊費（ホテル、宿泊施設など）
- 資格精算費（資格試験、受験料、認定費用など）
- その他経費（上記以外の業務関連経費）
