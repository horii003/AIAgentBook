## 2. 目的・役割定義

### 2.1 コンポーネントの目的
- AWS Strands SDK の FileSessionManager をラップし、セッションデータ（会話履歴・実行履歴・成果物参照）をローカルファイルシステムに永続化する
- セッション識別子（session_id）の生成・管理を担当する
- マルチエージェント構成（AG-001, AG-002, AG-003）でのセッション共有を実現する

### 2.2 主な責務
- **セッションID生成**: `session_{YYYYMMDD_HHMMSS}_{8桁ランダム文字列}` 形式のセッション識別子を生成する
- **FileSessionManager 生成**: セッションIDを使用して FileSessionManager インスタンスを生成する
- **セッションデータ永続化**: 会話履歴・実行履歴・成果物参照を storage/sessions/ に保存する
- **セッション再開**: 既存のセッションIDを使用して前回の会話履歴を復元する

### 2.3 非責務（やらないこと）
- **会話履歴の管理**: SlidingWindowConversationManager が担当する（AWS Strands SDK 提供）
- **invocation_state の管理**: AG-001 が生成し、各エージェントが参照する
- **セッション状態遷移**: main.py およびエージェントが担当する

### 2.4 必要性
- AWS Strands SDK の FileSessionManager を直接使用すると、セッションID生成・ディレクトリ管理が各エージェントに分散するため、一元管理が必要
- セッションIDの形式（`session_{YYYYMMDD_HHMMSS}_{8桁ランダム文字列}`）を統一するため、生成ロジックを集約する
- マルチエージェント構成でのセッション共有（AG-001 が生成したセッションIDを AG-002/AG-003 が使用）を実現するため

### 2.5 主要な設計意図
- **薄いラッパー**: FileSessionManager の薄いラッパーとして実装し、セッションID生成とディレクトリ管理のみを担当する
- **セッションID の一元生成**: main.py 起動時に AG-001 がセッションIDを生成し、invocation_state 経由で AG-002/AG-003 に伝播する
- **ファクトリ関数パターン**: 専門エージェント（AG-002/AG-003）はファクトリ関数内でセッションIDを使用して FileSessionManager を生成する

## 3. SessionManagerクラス

### 3.1 クラス概要
- **クラス名**: SessionManager
- **説明**: FileSessionManager のラッパークラス。セッションID生成と FileSessionManager インスタンス生成を担当する

### 3.2 クラス変数
- **_session_id**: str — 現在のセッション識別子（初期値: None）
- **_session_manager**: FileSessionManager — FileSessionManager インスタンス（初期値: None）
- **_storage_dir**: str — セッションデータの保存先ディレクトリ（初期値: "storage/sessions"）

**詳細**: 詳細設計書 セッションマネージャ詳細設計を参照

## 4. 主要メソッド

### 4.1 generate_session_id
- **目的**: `session_{YYYYMMDD_HHMMSS}_{8桁ランダム文字列}` 形式のセッション識別子を生成する
- **引数**: なし
- **戻り値**: str（セッション識別子）
- **フォーマット**: `session_{YYYYMMDD_HHMMSS}_{8桁ランダム文字列}`（例: `session_20260521_143022_a1b2c3d4`）

### 4.2 create_session_manager
- **目的**: セッションIDを使用して FileSessionManager インスタンスを生成する
- **引数**: session_id: str（セッション識別子）
- **戻り値**: FileSessionManager（AWS Strands SDK 提供）
- **デフォルトパス**: `storage/sessions/session_{session_id}/`

### 4.3 get_session_id
- **目的**: 現在のセッション識別子を返す
- **引数**: なし
- **戻り値**: str（セッション識別子）

### 4.4 get_session_manager
- **目的**: 現在の FileSessionManager インスタンスを返す
- **引数**: なし
- **戻り値**: FileSessionManager（AWS Strands SDK 提供）
- **フォーマット**: AG-001 の Agent() コンストラクタに渡す session_manager パラメータとして使用する

**詳細**: 詳細設計書 セッションマネージャ詳細設計を参照

## 5. セッション管理の仕組み

**システム設計書の方針**: artifacts/03_system-design/outputs/セッション管理方針.md を参照

### 5.1 セッション生成
1. main.py 起動時に SessionManager.generate_session_id() でセッション識別子を生成する
2. SessionManager.create_session_manager(session_id) で FileSessionManager インスタンスを生成する
3. AG-001 の Agent() コンストラクタに session_manager を渡す
4. AG-001 が invocation_state に session_id を設定して AG-002/AG-003 に伝播する

### 5.2 セッション更新
1. ユーザーとエージェントの会話ターンごとに FileSessionManager が会話履歴を自動保存する
2. ツール呼び出し結果（交通費計算結果・申請書生成結果等）も FileSessionManager に保存される
3. 申請書ファイルパスは生成完了時に FileSessionManager に保存される

### 5.3 セッション終了
1. ユーザーが終了を選択した場合、または対話回数が 30 回に達した場合にセッションを終了する
2. セッション状態を CLOSED または TERMINATED に更新する
3. セッションデータは storage/sessions/ に保持される（自動削除しない）

### 5.4 セッション再開
1. 既存のセッション識別子を使用して FileSessionManager を生成する
2. FileSessionManager が前回の会話履歴を自動復元する
3. 中断した申請フローから継続する

**詳細**: 詳細設計書 セッションマネージャ詳細設計を参照

## 6. マルチエージェント連携

### 6.1 セッション共有の仕組み

**AG-001（オーケストレーター）**:
1. main.py 起動時に SessionManager でセッション識別子を生成する
2. AG-001 の Agent() コンストラクタに FileSessionManager を渡す（self._session_manager）
3. AG-001 が invocation_state に session_id を設定する
4. AG-002/AG-003 を呼び出す際に invocation_state（session_id 含む）を渡す

**AG-002/AG-003（専門エージェント）**:
1. @tool(context=True) のツール関数内で tool_context.invocation_state から session_id を取得する
2. ファクトリ関数（_build_transportation_expense_agent / _build_general_expense_agent）内で session_id を使用して FileSessionManager を生成する
3. 生成した FileSessionManager を Agent() コンストラクタに渡す

### 6.2 会話履歴の共有範囲
- **共有される情報**: 会話履歴・ツール呼び出し結果・申請書ファイルパス（FileSessionManager 経由）
- **共有されない情報**: invocation_state（リクエスト単位で有効。セッションをまたがない）

## 11. 詳細設計への引き渡し事項

### 11.1 実装時の識別子
- **クラス名**: SessionManager（session/session_manager.py）

### 11.2 SDK実装時の注意事項
- AWS Strands SDK の FileSessionManager を使用する（strands.session.FileSessionManager）
- セッションデータの保存先ディレクトリは `storage/sessions/` とする（共通設定方針.md 準拠）
- セッション識別子の生成には datetime と secrets（または uuid）を使用する
- ログを出力するモジュールの先頭に `_logger = logging.getLogger(__name__)` を定義する（ログ出力方式設計.md 準拠）

### 11.3 モック／スタブ要否
- 単体テスト時は FileSessionManager をモックに差し替えること
- セッションデータの保存先は一時ディレクトリを使用すること

## 13. 制約事項・前提条件

### 13.1 技術的制約
- AWS Strands SDK（strands-agents >= 0.1.0）の FileSessionManager を使用すること
- Python 3.10 以上が必要
- storage/sessions/ ディレクトリへの書き込み権限が必要

### 13.2 業務的制約
- セッションデータは storage/sessions/ に保存する（プロジェクトルートからの相対パス）
- セッション識別子は `session_{YYYYMMDD_HHMMSS}_{8桁ランダム文字列}` 形式とする

### 13.3 前提条件
- storage/sessions/ ディレクトリが存在すること（存在しない場合は自動作成する）
- main.py 起動時にセッション識別子を生成し、AG-001 に渡すこと
