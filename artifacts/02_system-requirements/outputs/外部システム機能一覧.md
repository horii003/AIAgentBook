## 1. 外部連携システム一覧表（マスター）

本プロジェクトはCLIベースのスタンドアロンシステムであり、外部業務システム（ERP・申請システム等）との連携は行わない。
LLM推論基盤としてAmazon Bedrockを使用するが、これはフレームワーク（AWS Strands SDK）が内部的に管理するため、外部システムとして個別に定義しない。

## 2. インタフェース詳細定義（要件定義レベル）

### 2.1 IF-ID：IF-001
### API名：Bedrock InvokeModel API

---

#### 2.1.1 基本情報

| 項目 | 内容 |
|---|---|
| 外部システムID | EXT-001 |
| API提供元 | Amazon Web Services |
| 機能概要 | LLMモデル（Claude Sonnet）へのプロンプト送信と推論結果の受信 |
| 利用目的 | エージェントの自然言語処理・判断・生成 |
| 想定利用エージェント | AG-001, AG-002, AG-003 |

---

#### 2.1.2 I/F方式・非機能

| 項目 | 内容 |
|---|---|
| プロトコル | HTTPS |
| データ形式 | JSON |
| 認証方式 | AWS IAM（AWS_ACCESS_KEY_ID / AWS_SECRET_ACCESS_KEY） |
| レート制限 | AWSサービスクォータに依存 |
| SLA | 要件上未定義 |
| 冪等性 | 不要（LLM推論は非決定的） |

---

#### 2.1.3 リクエスト/レスポンス（概略）

##### リクエスト項目（要件定義レベル）
| 項目名 | 必須 | 説明 |
|---|---|---|
| model_id | ○ | 使用するLLMモデルID |
| messages | ○ | 会話履歴（システムプロンプト＋ユーザーメッセージ） |
| guardrail_id | — | ガードレールID（任意） |

##### レスポンス項目（要件定義レベル）
| 項目名 | 説明 |
|---|---|
| content | LLMの推論結果テキスト |
| stop_reason | 停止理由（end_turn / tool_use等） |

※ 詳細スキーマは基本設計／詳細設計へ

---

#### 2.1.4 エージェントツール化の可否評価

| 観点 | 評価 |
|---|---|
| 自動化適性 | 高 |
| 副作用リスク | 低（推論のみ） |
| 承認必須度 | 低 |
| ツール化可否 | 不可（Strands SDKが内部管理するため個別ツール化不要） |
