## 2. データ詳細定義（データID単位）

### 2.1 データID：DATA-001（データ名：経路テーブル）
- 種別：ファイル（マスタデータ）
- 概要：電車の出発地・目的地・運賃の対応テーブル
- 形式（拡張子/スキーマ）：JSON（.json）
- 保管場所：`data/train_fares.json`
- 更新頻度：随時（管理者が手動更新）
- 参照方法：ファイル読み込み（TOOL-001内で参照）
- データ分類・機密区分：公開
- マスキング要件：不要
- 保持期間：要件上未定義
- 品質要件：欠損なし・出発地/目的地/運賃の正確性高
- 監査要件：否

#### スキーマ（要件定義レベル：代表項目のみ）
| No | 項目名 | 型（概略） | 必須 | 説明 | 例 |
|---:|---|---|---|---|---|
| 1 | 出発地 | string | ○ | 出発駅名（正規化済み） | "東京" |
| 2 | 目的地 | string | ○ | 到着駅名（正規化済み） | "新宿" |
| 3 | 運賃 | number | ○ | 運賃（円） | 200 |

---

### 2.2 データID：DATA-002（データ名：固定運賃データ）
- 種別：ファイル（マスタデータ）
- 概要：バス・タクシー・飛行機の固定運賃データ
- 形式（拡張子/スキーマ）：JSON（.json）
- 保管場所：`data/fixed_fares.json`
- 更新頻度：随時（管理者が手動更新）
- 参照方法：ファイル読み込み（TOOL-001内で参照）
- データ分類・機密区分：公開
- マスキング要件：不要
- 保持期間：要件上未定義
- 品質要件：欠損なし・交通手段/運賃の正確性高
- 監査要件：否

#### スキーマ（要件定義レベル：代表項目のみ）
| No | 項目名 | 型（概略） | 必須 | 説明 | 例 |
|---:|---|---|---|---|---|

| 2 | 運賃 | number | ○ | 固定運賃（円） | 1000 |

## 3. AIエージェント向け追加項目（RAG/参照系データに推奨）

本プロジェクトではRAGを使用しない。申請ルール・業務ルールはエージェントのシステムプロンプトに直接埋め込む方式で実現する。

| 項目 | 内容 |
|---|---|
| 検索対象範囲 | 対象外（RAG不使用） |
| チャンク方針 | 対象外（RAG不使用） |
| メタデータ | 対象外（RAG不使用） |
| 根拠提示 | システムプロンプト内のルールIDを参照（Lv1〜Lv2） |
| 更新検知 | 対象外（RAG不使用） |
