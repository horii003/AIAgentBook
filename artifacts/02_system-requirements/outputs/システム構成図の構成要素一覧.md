# システム構成図の構成要素一覧

## 1. アーキテクチャ図

### 1.1 システム全体構成

| 要素ID | 要素名 | 種別 | 説明 | 技術スタック | 備考 |
|---|---|---|---|---|---|
| ARCH-01 | CLIインタフェース | フロントエンド | 一般社員がシステムと対話するCLIインタフェース | Python（標準入出力） | main.pyがエントリーポイント |
| ARCH-02 | 申請受付窓口エージェント | バックエンド | 申請内容を受け付け、申請種別を判断して専門エージェントへ委譲するオーケストレーター | AWS Strands SDK, Amazon Bedrock | AG-001 |
| ARCH-03 | 交通費精算申請エージェント | バックエンド | 交通費精算申請の情報収集・申請書生成を担当する専門エージェント | AWS Strands SDK, Amazon Bedrock | AG-002 |
| ARCH-04 | 経費精算申請エージェント | バックエンド | 経費精算申請の情報収集・申請書生成を担当する専門エージェント | AWS Strands SDK, Amazon Bedrock | AG-003 |
| ARCH-05 | 交通費計算ツール | バックエンド | 運賃データから交通費を計算する内部ツール | Python | TOOL-001 |
| ARCH-06 | 申請書生成ツール | バックエンド | 申請書テンプレートに情報を反映してExcelファイルを生成する内部ツール | Python, openpyxl | TOOL-002 |
| ARCH-07 | ローカルファイルストレージ | データベース | 電車経路テーブル・固定運賃データ・申請書テンプレート・生成済み申請書・セッションデータ・ログを保存するローカルファイルシステム | ファイルシステム | data/, template/, output/, storage/, logs/ |

## 6. 技術選定理由

### 6.1 主要技術の選定理由

| 技術名 | 選定理由 | 代替案 | 備考 |
|---|---|---|---|
| AWS Strands SDK | マルチエージェント構成・Agent as Toolパターン・セッション管理・フック機構を標準提供 | LangChain, AutoGen | 業務要件一覧に記載 |
| Amazon Bedrock（Claude Sonnet） | 高精度な自然言語理解・画像解析・日本語対応 | OpenAI GPT, Google Gemini | 業務要件一覧に記載 |
| Python 3.10以上 | 型ヒント・match文対応、Strands SDKの対応言語 | — | 業務要件一覧に記載 |
| openpyxl | ExcelファイルのPython操作ライブラリ | xlwt, xlrd | 申請書テンプレートがExcel形式のため |
| Pydantic | データバリデーション・型安全性の確保 | marshmallow | 業務要件一覧に記載 |

### 6.2 技術的制約事項

| 制約ID | 制約内容 | 影響範囲 | 対応方針 |
|---|---|---|---|
| CONST-01 | Amazon Bedrockへのアクセスにはインターネット接続が必要 | 全エージェント | AWS認証情報の適切な管理 |
| CONST-02 | 申請書の提出はAIが行わず必ず人が実行する（BRL-06） | AG-002, AG-003 | 申請書生成後は人が確認・提出する設計 |
