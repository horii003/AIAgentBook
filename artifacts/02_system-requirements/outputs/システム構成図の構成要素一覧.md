## 6. 技術選定理由

### 6.1 主要技術の選定理由

| 技術名 | 選定理由 | 代替案 | 備考 |
|---|---|---|---|
| Python 3.10以上 | 型ヒント・match文対応。Strands SDKの要件 | — | |
| AWS Strands SDK | マルチエージェント・Agent as Tools・HumanApprovalHookを標準サポート | LangChain, AutoGen | |
| Amazon Bedrock（Claude Sonnet） | 高精度な日本語処理・ガードレール機能 | OpenAI GPT | |
| Pydantic v2 | 型安全なデータバリデーション | dataclasses | |
| openpyxl | Excelファイル生成 | xlwt | |

### 6.2 技術的制約事項

| 制約ID | 制約内容 | 影響範囲 | 対応方針 |
|---|---|---|---|
| CONST-01 | Amazon Bedrockはap-northeast-1リージョンで使用 | LLM推論全般 | AWS_DEFAULT_REGIONで設定 |
| CONST-02 | CLIベースのスタンドアロン実行（Webサーバー不使用） | インタフェース | main.pyで標準入出力を使用 |
| CONST-03 | RDB不使用（ファイルベース管理） | データ永続化 | JSONファイル・Excelファイル・テキストログで管理 |
| CONST-04 | ユーザー入力文字数上限：500文字 | CLIインタフェース | 500文字超過時は再入力を促す |
| CONST-05 | 対話回数上限：30回 | 全エージェント | LoopControlHookで制御 |
