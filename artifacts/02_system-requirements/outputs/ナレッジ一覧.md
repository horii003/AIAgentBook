## 2. ナレッジ詳細定義（ナレッジID単位）

### 2.1 ナレッジID：KN-001
### ナレッジ名：申請種別判断ルール

---

## 2.1.1 ナレッジ概要
- 申請内容テキストから交通費精算申請・経費精算申請のいずれかを判断するためのルール
- 業務上の位置づけ：必須（申請種別判断の唯一の根拠）

---

## 2.1.2 ナレッジ種別・特性

| 項目 | 内容 |
|---|---|
| 種別 | 業務ルール |
| 判断の性質 | 決定的 |
| 判断の粒度 | 厳密 |
| 競合時の優先度 | 高 |

---

## 2.1.3 利用範囲・制約

| 項目 | 内容 |
|---|---|
| 利用可能エージェント | AG-001 |
| 利用可能自律度 | 最大 Lv5 |
| 利用禁止ケース | なし |
| 人判断が必要な条件 | 判断不能時（JD-03でNG）はユーザーに選択肢を提示 |

---

## 2.1.4 判断拘束力

- 本ナレッジの拘束力：**強**
- このナレッジに定義されたルールに必ず従う。ルールに記載のない内容は推測・補完しない

---

## 2.1.5 入力・参照条件

| No | 条件項目 | 内容 |
|---:|---|---|
| 1 | 前提条件 | 申請内容テキストが入力済みであること |
| 2 | 判断条件 | BRL-01（交通費精算該当）、BRL-02（経費精算該当）、BRL-03（判断不能）、BRL-05（対象外） |
| 3 | 参照トリガー | 申請内容テキスト受付後、申請種別判断時 |

---

## 2.1.6 ナレッジ構造定義

### b. ルール型（条件定義）
```text
IF 申請内容に「業務上の交通費精算（電車・バス・タクシー・飛行機）」が含まれる
THEN 交通費精算申請を案内

IF 申請内容に「業務上の経費精算（交通費を除く）」が含まれる
THEN 経費精算申請を案内

IF 上記いずれにも該当せず、精算に関連する
THEN ユーザーに「交通費精算申請」「経費精算申請」の選択肢を提示

IF 交通費精算・経費精算のいずれにも関連しない
THEN 担当者へのエスカレーションを案内
```

## 3. ナレッジ運用上の重要な整理軸（設計標準）

### ① 「ナレッジ＝RAG」ではない
- 本プロジェクトではRAGは使用しない
- ナレッジはシステムプロンプトに直接埋め込む方式で提供する
- エージェントの判断に使用するナレッジのみを列挙している

### ② 判断拘束力を必ず定義
- 強（従う）：KN-001〜KN-004（申請ルール・業務ルール）
- 中（原則）：KN-005（経費区分判断基準、品目によっては曖昧なケースあり）
- 弱（参考）：該当なし

### ③ 自律度と必ずペアで管理
- 全ナレッジ：Lv5まで使用可（自動実行レベルで参照可能）
- 判断不能時のみ人の介入が必要（Lv2：提案レベルに降格）
