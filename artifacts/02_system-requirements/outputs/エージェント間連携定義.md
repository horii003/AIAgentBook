# エージェント間連携一覧（協調・委譲・イベント）

## 1. 連携ポリシー（概要）
- 連携方式：同期（リクエスト/レスポンス）
- 連携単位：タスク（申請種別確定後に専門エージェントへ委譲）
- 失敗時方針：エラーメッセージを表示し人へエスカレーション
- 相互呼び出し制約：循環呼び出し禁止、最大委譲深度：1（オーケストレーター→専門エージェント）、タイムアウト：要件上未定義

## 3. 連携種別の定義（記入ルール）

### 3.1 協調（Collaboration）
- 本プロジェクトでは該当なし

### 3.2 委譲（Delegation）
- 目的：オーケストレーター（AG-001）が申請種別確定後に専門エージェント（AG-002/AG-003）へ申請書作成タスクを委譲する
- パターン：Agent as Tool（Strands SDK標準パターン）

### 3.3 イベント（Event-driven）
- 本プロジェクトでは該当なし

---

## 4. 連携インタフェース定義

### 4.1 連携ID：LNK-001

#### 4.1.1 トリガー条件
- 発火条件：AG-001が申請種別を「経費精算申請」と判断（またはユーザーが選択）した時
- 事前条件：申請種別判断結果（D-003）が「経費精算申請」として確定していること
- 事後条件：経費精算申請書ファイルが生成され社員に提示される、またはエラーメッセージが返される

#### 4.1.2 入力（invocation_state経由）
- 申請者名（user_name）
- 申請日（request_date）
- セッションID（session_id）

#### 4.1.3 出力
- 成功時：申請書生成完了メッセージ（ファイルパスを含む）
- 失敗時：エラーメッセージ（原因を含む）
