# エージェント間連携一覧（協調・委譲・イベント）

## 1. 連携ポリシー（概要）
- 連携方式：同期（リクエスト/レスポンス）
- 連携単位：タスク（業務単位）
- 失敗時方針：エラーメッセージを返しユーザーへ通知
- 相互呼び出し制約：循環呼び出し禁止、最大委譲深度：1（AG-001 → AG-002/AG-003）、タイムアウト：要件上未定義

## 3. 連携種別の定義（記入ルール）

### 3.1 協調（Collaboration）
- 目的：複数エージェントが並行/分担して成果を統合
- 本プロジェクトでの使用：なし

### 3.2 委譲（Delegation）
- 目的：起点エージェントがサブタスクを丸投げして結果を受け取る
- 本プロジェクトでの使用：LNK-001, LNK-002（AG-001 → AG-002/AG-003）

### 3.3 イベント（Event-driven）
- 目的：状態変化を発火し、購読側が処理
- 本プロジェクトでの使用：なし

## 4. 連携インタフェース定義

### 4.1 連携ID：LNK-001（AG-001 → AG-002）

#### 4.1.1 トリガー条件
- 発火条件：AG-001が申請種別を「交通費精算申請」と確定した時
- 事前条件：申請者名が入力済みであること
- 事後条件：AG-002が交通費精算申請書作成処理を完了し結果を返す

#### 4.1.2 入力（概念）
| 入力項目 | 説明 | 必須/任意 |
|---------|------|-----------|
| query | AG-001からの指示テキスト（申請内容・申請者名を含む） | 必須 |
| invocation_state.user_name | 申請者名 | 必須 |
| invocation_state.request_date | 申請日 | 必須 |
| invocation_state.session_id | セッションID（AG-002のセッション管理に使用） | 必須 |

#### 4.1.3 出力（概念）
| 出力項目 | 説明 |
|---------|------|
| 処理結果テキスト | 申請書生成完了メッセージ（ファイルパス含む）またはエラーメッセージ |

#### 4.1.4 エラー時動作
| エラーパターン | 動作 |
|---|---|
| LoopLimitError（ループ上限到達） | エラーメッセージを返しAG-001経由でユーザーへ通知 |
| Exception（予期しないエラー） | エラーメッセージを返しAG-001経由でユーザーへ通知 |
