# エージェント間連携一覧（協調・委譲・イベント）

## 1. 連携ポリシー（概要）
- 連携方式：同期（リクエスト/レスポンス）
- 連携単位：タスク（申請種別確定後に専門エージェントへ委譲）
- 失敗時方針：エラーを返す / 人へエスカレーション
- 相互呼び出し制約：循環呼び出し禁止、最大委譲深度：1（AG-001 → AG-002 または AG-003）、タイムアウト：要件上未定義

## 3. 連携種別の定義（記入ルール）

### 3.1 協調（Collaboration）
- 目的：複数エージェントが**並行/分担**して成果を統合
- 本プロジェクトでの使用：なし

### 3.2 委譲（Delegation）
- 目的：起点エージェントがサブタスクを**丸投げ**して結果を受け取る
- 本プロジェクトでの使用：LNK-001（AG-001 → AG-002）、LNK-002（AG-001 → AG-003）

### 3.3 イベント（Event-driven）
- 目的：状態変化を発火し、購読側が処理
- 本プロジェクトでの使用：なし

## 4. 連携インタフェース定義（最小セット）

### 4.1 連携ID：LNK-001

#### 4.1.1 トリガー条件
- 発火条件：AG-001が申請種別を交通費精算申請として確定した時
- 事前条件（Pre-condition）：申請種別が交通費精算申請として確定していること
- 事後条件（Post-condition）：交通費精算申請書ファイルが生成されること

#### 4.1.2 入力スキーマ（構造化推奨）
```json
{
  "query": "string（申請内容・ユーザーへの質問）",
  "invocation_state": {
    "user_name": "string",
    "request_date": "string（YYYY-MM-DD）",
    "session_id": "string"
  }
}
```
