## 3. 評価の実施方法

### 3.1 開発段階における評価

| フェーズ | 実施方法 | 対象観点 |
|---------|---------|---------| 
| 開発初期 | 人手評価 | 妥当性・有用性 |
| 開発中期 | 自動評価の段階的導入 | 妥当性・有用性 |
| リリース前 | 全テストケースの自動実行 | 妥当性・有用性（必須ゲート） |

### 3.2 回帰評価

| トリガー | 実施内容 | 合否基準 |
|---------|---------|---------| 
| プロンプト変更時 | 全テストケースを再実行 | 変更前と同等以上のスコア |
| ナレッジ変更時 | 全テストケースを再実行 | 変更前と同等以上のスコア |
| モデルバージョン変更時 | 全テストケースを再実行 | 各MET-xxxの閾値を満たすこと |

### 3.3 運用段階における継続的評価

- リリース後も品質は変動し続けるため、評価を運用サイクルに組み込む
- 定期的に全テストケースの回帰評価を実施する

# 評価指標定義書

## 評価指標ID：MET-001
## 評価指標名：申請種別判断正確率

### 1. 基本情報

| 項目 | 内容 |
|---|---|
| 評価観点 | 妥当性 |
| 対象エージェント | AG-001 |
| 対象会話フロー | CF-001 |

### 2. 指標の目的・意味

- 申請内容から交通費精算申請・経費精算申請のいずれかを正しく判断できているかを評価する
- 申請種別の誤判断は申請ミス・差し戻しに直結するため、高い正確率が必要
- スコアが低い場合はシステムプロンプトの申請ルール定義を見直す

### 3. 指標定義

| 項目 | 内容 |
|---|---|
| 指標種別 | 定量（LLM-as-Judge二値） |
| 定義式 | 正しく判断されたケース数 / 全テストケース数 |
| 評価単位 | テストケース |
| スコア範囲 | 0.0〜1.0 |
| 高いほど良い | Yes |

### 4. 評価方法・測定手順

| 項目 | 内容 |
|---|---|
| 評価方法 | 交通費精算申請・経費精算申請・判断不能の各ケースを含むテストセットを用意し、ToolSelectionAccuracyEvaluatorで申請種別に応じた正しいエージェントが選択されるかを判定する |
| 実施手順 | 1. テストケースを実行 2. ツール呼び出し履歴を取得 3. ToolSelectionAccuracyEvaluatorで二値判定 |
| 評価者 | システム（LLM） |

### 5. 合格基準・閾値

| 項目 | 内容 |
|---|---|
| 目標値 | 0.95以上 |
| 最低許容値 | 0.9以上 |
| 不合格時対応 | システムプロンプトの申請ルール定義を見直す |

### 6. 評価頻度

| フェーズ | 実施有無 |
|---|---|
| 開発中 | ○ |
| リリース前 | ○（必須ゲート） |
| 運用中 | ○ |

## 評価指標ID：MET-002
## 評価指標名：経費申請ゴール達成率

### 1. 基本情報

| 項目 | 内容 |
|---|---|
| 評価観点 | 有用性 |
| 対象エージェント | AG-001, AG-002, AG-003 |
| 対象会話フロー | CF-001〜CF-005 |

### 2. 指標の目的・意味

- 経費精算申請書を作成・提出できるというビジネスゴールが達成されるかを評価する
- ゴール達成率が低い場合は情報収集フローや申請書生成ロジックを見直す

### 3. 指標定義

| 項目 | 内容 |
|---|---|
| 指標種別 | 定量（LLM-as-Judge二値） |
| 定義式 | ゴール達成セッション数 / 全テストセッション数 |
| 評価単位 | セッション |
| スコア範囲 | 0.0〜1.0 |
| 高いほど良い | Yes |

### 4. 評価方法・測定手順

| 項目 | 内容 |
|---|---|
| 評価方法 | ActorSimulatorを使用したマルチターン評価。経費精算申請書ファイルが生成されることをゴールとして設定する |
| 実施手順 | 1. テストシナリオを実行 2. 申請書ファイルの生成を確認 3. GoalSuccessRateEvaluatorで判定 |
| 評価者 | システム（LLM） |

### 5. 合格基準・閾値

| 項目 | 内容 |
|---|---|
| 目標値 | 0.9以上 |
| 最低許容値 | 0.8以上 |
| 不合格時対応 | 情報収集フロー・申請書生成ロジックを見直す |

| 運用中 | ○ |
