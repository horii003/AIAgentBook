## 1. 概要

### 1.1 コンポーネントの目的

各エージェントが参照する業務ルール・ポリシーを Python モジュールとして定義する。ビジネスルール値（期限月数・金額閾値等）は `settings.py` で一元管理し、ポリシー関数の引数として受け取って動的に展開する。

### 1.2 ファイル構成

| ファイル | 内容 | 利用エージェント |
|--------|------|--------------|
| `knowledge/transportation_policies.py` | 交通費精算申請の業務ルール | AG-002 |
| `knowledge/receipt_policies.py` | 経費精算申請の業務ルール | AG-003 |

### 2.1 基本情報

| 項目 | 内容 |

| 利用エージェント | AG-002（交通費精算申請エージェント） |
| 利用タイミング | `prompt/prompt_transportation_expense.py` からシステムプロンプト生成時 |

### 2.2 関数定義

#### get_transportation_policies

**シグネチャ**:
```python
def get_transportation_policies(deadline_months: int, approval_threshold: int) -> str:
```

**引数**:
- `deadline_months` (`int`): 申請期限（月数）。`settings.transportation.deadline_months` から取得
- `approval_threshold` (`int`): 上長承認閾値（円）。`settings.transportation.approval_threshold` から取得

**戻り値**:
- `str`: 業務ルールテキスト

**ポリシーテキスト**:
```python
def get_transportation_policies(deadline_months: int, approval_threshold: int) -> str:
    return f"""
- BRL-07: 移動情報は一区間ずつ収集する（出発地・目的地・交通手段・移動日・業務目的）
- BRL-08: 運賃は calculate_transport_fare ツールで自動計算する
- BRL-09: 申請期限は経費発生日から{deadline_months}ヶ月以内。超過時は申請不可を案内して処理を終了する
- BRL-10: 交通費合計が{approval_threshold:,}円を超える場合は上長承認欄を追加する（needs_approval=True）
- BRL-11: 電車の場合は駅名を正規化してから calculate_transport_fare ツールに渡す
- BRL-12: 業務目的は必須。未記載時は入力を促す
"""
```

## 5. 利用方法

### 5.1 システムプロンプト生成時の呼び出し

```python
# prompt/prompt_transportation_expense.py
from knowledge.transportation_policies import get_transportation_policies
from config.settings import settings

def get_transportation_expense_system_prompt(
    applicant_name: str,
    application_date: str,
    deadline: str,
) -> str:
    policies = get_transportation_policies(
        deadline_months=settings.transportation.deadline_months,
        approval_threshold=settings.transportation.approval_threshold,
    )
    return _TRANSPORTATION_EXPENSE_TEMPLATE.format(
        applicant_name=applicant_name,
        application_date=application_date,
        deadline=deadline,
        transportation_policies=policies,
    )
```

## 7. テスト観点

### 7.1 機能テスト
- `get_transportation_policies(3, 10000)` が正しいポリシーテキストを返すこと
- `get_receipt_policies(3, 5000)` が正しいポリシーテキストを返すこと
- 設定値が変更された場合にポリシーテキストに反映されること

### 7.2 境界値テスト
- `deadline_months=1` の場合に正しく展開されること
- `approval_threshold=0` の場合に正しく展開されること
