## 1. 概要

### 1.1 コンポーネントの目的

Amazon Bedrock ガードレールとアプリケーション実装（LoopControlHook / HumanApprovalHook）を組み合わせて、全エージェントに対するセキュリティ・業務ルール制御を実現する。

### 1.2 ガードレール構成

| 層 | 実現方式 | 担当コンポーネント |
|----|---------|----------------|
| 入力ガード | Amazon Bedrock ガードレール | `config/model_config.py`（BedrockModel 設定） |
| 出力ガード | Amazon Bedrock ガードレール | `config/model_config.py`（BedrockModel 設定） |
| 実行ガード（ループ制限） | LoopControlHook | `handlers/loop_control_hook.py` |
| 実行ガード（申請書生成前確認） | HumanApprovalHook | `handlers/human_approval_hook.py` |

---

## 2. Amazon Bedrock ガードレール設定

### 2.1 設定パラメータ

| 設定項目 | 管理方式 | 設定値 |
|--------|---------|-------|
| ガードレールID | 環境変数（`GUARDRAIL_ID`） | Amazon Bedrock で作成後に設定 |
| ガードレールバージョン | 環境変数（`GUARDRAIL_VERSION`） | `DRAFT` |
| ガードレールトレース | `config/model_config.py` | `"enabled"` |

### 2.2 BedrockModel 設定

```python
# config/model_config.py
BedrockModel(
    model_id=cls.DEFAULT_MODEL_ID,
    guardrail_id=cls.GUARDRAIL_ID,           # 環境変数 GUARDRAIL_ID から取得
    guardrail_version=cls.GUARDRAIL_VERSION,  # 環境変数 GUARDRAIL_VERSION から取得
    guardrail_trace="enabled"                 # 全エージェント共通
)
```

### 2.3 ガードレール要件と実装方式

| ガードレールID | 機能カテゴリ | 実現方式 | 検知時の処理 |
|-------------|-----------|---------|-----------|
| GRD-001 | 入力ガード（プロンプトインジェクション対策） | Amazon Bedrock ガードレール | 入力をブロックし、エラーメッセージを返す |
| GRD-002 | 入力ガード（有害コンテンツフィルタ: MEDIUM） | Amazon Bedrock ガードレール | 入力をブロックし、エラーメッセージを返す |
| GRD-003 | ルール検証（申請書提出禁止） | HumanApprovalHook | 処理を停止し、人に提出を促す |
| GRD-004 | ルール検証（申請種別制限） | Amazon Bedrock ガードレール（トピック拒否） | 対応不可を案内する |
| GRD-005 | 出力ガード（機密情報保護: PII マスキング） | Amazon Bedrock ガードレール | 出力をマスキングする |
| GRD-006 | 出力ガード（有害コンテンツフィルタ: MEDIUM） | Amazon Bedrock ガードレール | 出力をブロックし、エラーメッセージを返す |
| GRD-007 | 実行ガード（ループ制限: 最大10回） | LoopControlHook | LoopLimitError を raise する |
| GRD-008 | 実行ガード（申請書生成前確認） | HumanApprovalHook | 人の承認を待つ |
| GRD-009 | 入力ガード（PII: CVV フィルタ） | Amazon Bedrock ガードレール | 入力をブロックし、エラーメッセージを返す |
| GRD-010 | 出力ガード（PII: CVV フィルタ） | Amazon Bedrock ガードレール | 出力をブロックし、エラーメッセージを返す |

## 4. CloudFormation テンプレート

### 4.1 ファイルパス

`guardrails/guardrails_cloudformation.yaml`

### 4.2 テンプレート概要

```yaml
AWSTemplateFormatVersion: '2010-09-09'
Description: Amazon Bedrock Guardrails for AI Agent Application

Resources:
  AgentGuardrail:
    Type: AWS::Bedrock::Guardrail
    Properties:
      Name: ai-agent-guardrail
      Description: AIエージェント申請システム用ガードレール
      BlockedInputMessaging: "入力内容が安全ポリシーに違反しています。別の内容で再度お試しください。"
      BlockedOutputsMessaging: "出力内容が安全ポリシーに違反しています。申請管理部門にお問い合わせください。"

      # コンテンツフィルタリング（入力・出力）
      ContentPolicyConfig:
        FiltersConfig:
          - Type: HATE
            InputStrength: MEDIUM
            OutputStrength: MEDIUM
          - Type: INSULTS
            InputStrength: MEDIUM
            OutputStrength: MEDIUM
          - Type: SEXUAL
            InputStrength: MEDIUM
            OutputStrength: MEDIUM
          - Type: VIOLENCE
            InputStrength: MEDIUM
            OutputStrength: MEDIUM
          - Type: MISCONDUCT
            InputStrength: MEDIUM
            OutputStrength: MEDIUM
          - Type: PROMPT_ATTACK
            InputStrength: HIGH
            OutputStrength: NONE

      # 機密情報保護（PII マスキング）
      SensitiveInformationPolicyConfig:
        PiiEntitiesConfig:
          - Type: NAME
            Action: ANONYMIZE
          - Type: EMAIL
            Action: ANONYMIZE
          - Type: PHONE
            Action: ANONYMIZE
          - Type: CREDIT_DEBIT_CVV
            Action: BLOCK

      # トピック拒否（申請種別制限）
      TopicPolicyConfig:
        TopicsConfig:
          - Name: out-of-scope-requests
            Definition: "交通費精算申請・経費精算申請以外の申請種別（例: 有給申請、採用申請、設備申請等）"
            Examples:
              - "有給休暇を申請したい"
              - "採用の申請をしたい"
            Type: DENY

Outputs:
  GuardrailId:
    Description: Guardrail ID
    Value: !GetAtt AgentGuardrail.GuardrailId
  GuardrailArn:
    Description: Guardrail ARN
    Value: !GetAtt AgentGuardrail.GuardrailArn
```

## 5. 環境変数設定

### 5.1 .env.template への追加

```
GUARDRAIL_ID=
GUARDRAIL_VERSION=DRAFT
```

### 5.2 設定手順

1. `guardrails/guardrails_cloudformation.yaml` を使用して Amazon Bedrock ガードレールを作成する
2. 作成されたガードレール ID を `.env` の `GUARDRAIL_ID` に設定する
3. `GUARDRAIL_VERSION` は `DRAFT` のままで動作する

## 6. テスト観点

### 6.1 機能テスト
- プロンプトインジェクション攻撃が検出・ブロックされること（GRD-001）
- 有害コンテンツが検出・ブロックされること（GRD-002, GRD-006）
- PII（氏名・メールアドレス等）が出力でマスキングされること（GRD-005）
- CVV が入力・出力でブロックされること（GRD-009, GRD-010）
- 定義外の申請種別が拒否されること（GRD-004）
- LoopControlHook が 10 回のループ上限を正しく制御すること（GRD-007）
- HumanApprovalHook が申請書生成前に承認を求めること（GRD-008）

### 6.2 異常系テスト
- ガードレール ID が未設定の場合に警告ログが出力されること
