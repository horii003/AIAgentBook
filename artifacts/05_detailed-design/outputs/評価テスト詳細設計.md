## 1. 概要

### 1.1 テスト設計の目的

各コンポーネントの詳細設計書に記載されたテスト観点を集約・体系化し、評価指標一覧（EVAL-001〜EVAL-007）を満たす実行可能なテストケースを設計する。単体・統合・E2E の3層構造でテストを整理し、pytest による自動化を基本方針とする。

### 1.2 テスト対象範囲

| テスト層 | 対象コンポーネント | 実施方式 |
|---------|--------------|--------|
| 単体テスト | ツール（TOOL-001, TOOL-002）、データモデル、ハンドラー（ErrorHandler, LoopControlHook, HumanApprovalHook）、セッションマネージャ、ナレッジ・業務ルール | pytest（自動） |
| 統合テスト | エージェント連携（AG-001→AG-002/AG-003）、ツール呼び出しフロー、セッション永続化 | pytest（自動） |
| E2Eテスト | 全会話フロー（CF-001〜CF-003）、評価指標検証 | strands-agents-evals（自動）+ 手動 |

### 1.3 テスト環境

| 項目 | 値 |
|------|-----|
| Python バージョン | 3.10以上 |
| モデルID | `jp.anthropic.claude-sonnet-4-5-20250929-v1:0` |
| テストフレームワーク | `pytest >= 7.4.0`, `pytest-cov >= 4.1.0` |
| 評価フレームワーク | `strands-agents-evals >= 0.1.0` |

## 2. テスト全体構成

### 2.1 テスト層とカバレッジ方針

| テスト層 | カバレッジ目標 | 重点観点 |
|---------|------------|--------|
| 単体テスト | 主要ロジック網羅 | バリデーション・エラーハンドリング・境界値 |
| 統合テスト | 主要連携フロー網羅 | エージェント間連携・ツール呼び出し・セッション永続化 |
| E2Eテスト | 全評価シナリオ（6分類） | 評価指標 EVAL-001〜EVAL-007 の達成確認 |

### 2.2 テストデータ・モック方針

| コンポーネント | モック方針 | 理由 |
|------------|---------|------|
| Amazon Bedrock API | `unittest.mock.patch` でモック | テストコストの削減・再現性確保 |
| FileSessionManager | 一時ディレクトリ（`tmp_path` フィクスチャ）を使用 | ファイルI/O の実際の動作を検証 |
| 運賃データ JSON | テスト用 JSON ファイルを `data/` に配置 | 実際のデータ読み込みを検証 |
| Excel テンプレート | テスト用テンプレートを `template/` に配置 | 実際のファイル生成を検証 |
| ユーザー入力（input()） | `unittest.mock.patch("builtins.input")` でモック | HumanApprovalHook のテスト |

## 6. 評価指標検証設計

| 評価指標ID | 指標名 | 測定方法 | 合否基準（閾値） | 測定ツール |
|----------|-------|--------|-------------|---------|
| EVAL-001 | 申請種別判断正確性 | 各シナリオ分類（正常系・分岐系・対象外）のテストケースに対する正答率を測定 | 判断誤り率 0%（全テストケースで正解） | strands-agents-evals + pytest |
| EVAL-002 | 経費区分判断正確性 | 品目と経費区分の対応テストケースに対する正答率を測定 | 要件上未定義（全テストケースで正解を目標） | strands-agents-evals + pytest |
| EVAL-003 | 交通費計算正確性 | 運賃データに基づく計算結果と期待値の一致率を測定 | 計算誤り率 0%（全テストケースで正解） | pytest（TC-UNIT-001〜004） |
| EVAL-004 | 申請書生成完全性 | 生成された Excel ファイルの全必須項目の記入状況を確認 | 記入漏れ率 0%（全必須項目が記入済み） | pytest（TC-UNIT-020〜022） |
| EVAL-005 | 業務ルール遵守率 | 全 BRL ルールに対応するテストケースの合格率を測定 | 違反率 0%（全テストケースで正解） | pytest（TC-E2E-005, TC-E2E-006） |
| EVAL-006 | エスカレーション適切性 | 対象外申請に対するエスカレーション案内の正答率を測定 | 誤判定率 0%（全テストケースで正解） | pytest（TC-E2E-004） |
| EVAL-007 | 会話完了率 | 全 E2E シナリオの正常完了率を測定 | 要件上未定義（全シナリオの完了を目標） | strands-agents-evals（TC-E2E-001〜006） |

### 7.3 モック方針

| モック対象 | モック方式 | 理由 |
|----------|---------|------|
| Amazon Bedrock API | `unittest.mock.patch("strands.models.BedrockModel.__call__")` | テストコストの削減・再現性確保 |
| `builtins.input` | `unittest.mock.patch("builtins.input")` | HumanApprovalHook のユーザー入力をモック化 |
| FileSessionManager | `tmp_path` フィクスチャで一時ディレクトリを使用 | 実際のファイルI/O を検証しつつ副作用を防止 |
| 運賃データ JSON | テスト用 JSON ファイルを `tests/fixtures/` に配置 | 実際のデータ読み込みを検証 |
