## 1. 概要

### 1.1 コンポーネントの目的

AG-001から委譲された交通費精算申請を完結させる。移動情報を対話で収集し、交通費を自動計算して申請書ドラフトを提示する。ユーザーのOK選択後に交通費精算申請書（Excel）を生成する。

### 1.2 主要な責務

- **移動情報収集**: 移動日・出発地・目的地・交通手段・業務目的を1区間ずつ対話で収集する（BRL-04）
- **交通費計算**: TOOL-001（calculate_transport_fare）を呼び出して交通費を自動計算する（BRL-05）
- **申請期限チェック**: 移動日から申請日まで90日超の場合に警告を提示する（BRL-06）
- **上長承認要否通知**: 交通費合計が10,000円超の場合に通知する（BRL-07）
- **申請書ドラフト提示**: 収集情報をテキストで整理・提示する（ツール呼び出しなし）
- **申請書生成**: ユーザーのOK選択後にTOOL-002（generate_transport_form）を呼び出す

### 2.3 ビジネスロジック

#### 2.3.1 システムプロンプト

##### システムプロンプトの生成方式

| 項目 | 内容 |
|------|------|
| 生成方式 | 動的生成関数 `get_transport_system_prompt(applicant_name, application_date, deadline_date)` |
| 動的生成の理由 | BRL-06（申請期限チェック）でLLMが移動日と申請日を比較するため、実行時の申請日・申請期限日をプロンプトに埋め込む必要がある |
| 妥当性チェックルール参照先 | `knowledge/transportation_policies.py` の `get_transportation_policies()` 関数を呼び出してプロンプトに組み込む |

##### 動的に埋め込む情報
- `applicant_name`: 申請者名
- `application_date`: 申請日（YYYY-MM-DD）
- `deadline_date`: 申請期限（application_dateから90日前の日付、YYYY-MM-DD）

##### プロンプト全文

```
あなたは交通費精算申請エージェントです。申請者「{applicant_name}」の交通費精算申請を処理します。
本日の申請日: {application_date}
申請可能な移動日の下限: {deadline_date}（申請日から90日以内の移動のみ申請可能）

【役割】
1. 移動情報（移動日・出発地・目的地・交通手段・業務目的）を1区間ずつ収集する
2. 各区間の交通費をcalculate_transport_fareツールで自動計算する
3. 申請期限・上長承認要否を確認し、必要に応じて警告・通知を行う
4. 申請書ドラフトをテキストで提示し、ユーザーの確認を求める
5. ユーザーのOK選択後にgenerate_transport_formツールで申請書を生成する

【移動情報の収集ルール（BRL-04）】
- 1区間ずつ収集する（複数区間がある場合は繰り返す）
- 必須項目: 移動日（YYYY-MM-DD形式）・出発地・目的地・交通手段・業務目的
- 対応交通手段: 電車・バス・タクシー・飛行機（BRL-03）
- 業務目的は必須（BRL-09）

【申請期限チェック（BRL-06）】
- 移動日が {deadline_date} より前の場合: 「申請期限（90日）を超過しています。申請を続けますか？」と警告を提示する
- 警告後もユーザーが希望する場合は処理を継続する

【上長承認要否（BRL-07）】
- 交通費合計が10,000円を超える場合: 「交通費合計が10,000円を超えるため、上長承認が必要です。」と通知する（処理継続）

【申請書ドラフト提示】
- 収集した全移動情報と交通費合計をテキストで整理して提示する
- ツール（generate_transport_form）は呼び出さない
- 「以上の内容で申請書を作成してよろしいですか？（OK / 修正 / キャンセル）」と確認する

【申請書生成】
- ユーザーが「OK」を選択した場合のみgenerate_transport_formを呼び出す
- 「修正」の場合は移動情報収集へ戻る
- 「キャンセル」の場合は処理を中断する

【エラー発生時】
- ツールエラー・システム障害等が発生した場合は、エラー内容を要約して呼び出し元エージェントに返す
```

##### プロンプト設計の意図

| 要素 | 意図 |
|------|------|
| 動的日付埋め込み | BRL-06の申請期限チェックをLLMが正確に判断できるようにする |
| ドラフト提示の明示 | ツール呼び出しなしでテキスト提示することを明示し、HumanApprovalHookが1回のみ発火するようにする |
| エラー時の振る舞い | システム系エラーをAG-001へ要約返却することを明示する |

#### 2.3.2 申請ルール

| ルール | 条件 | 対応 |
|--------|------|------|
| BRL-03 | 対応外交通手段 | バリデーションエラーとして再入力を促す |
| BRL-04 | 移動情報収集 | 1区間ずつ収集する |
| BRL-05 | 交通費計算 | calculate_transport_fareを呼び出す |
| BRL-06 | 申請期限超過（90日超） | 警告提示後に処理継続 |
| BRL-07 | 交通費合計10,000円超 | 上長承認要否を通知（処理継続） |
| BRL-08 | 駅名正規化 | validate_station_nameで自動正規化 |
| BRL-09 | 業務目的必須 | バリデーションエラーとして再入力を促す |

#### 2.3.3 処理フロー

```
開始（AG-001からの委譲受付）
  ↓
[ステップ1] invocation_stateから申請者名・申請日・session_idを取得
  ↓
[ステップ2] FileSessionManagerを生成（session_idを使用）
  ↓
[ステップ3] 動的システムプロンプト生成
  - get_transport_system_prompt(applicant_name, application_date, deadline_date)
  ↓
[ステップ4] エージェントインスタンス生成
  - _build_transport_agent(session_id, applicant_name, application_date, deadline_date)
  ↓
[ステップ5] エージェント呼び出し
  - agent(query, invocation_state={"applicant_name": ..., "application_date": ...})
  ↓
[エージェント内部処理]
  - 移動情報収集 → calculate_transport_fare呼び出し → 申請期限チェック → ドラフト提示 → generate_transport_form呼び出し
  ↓
[ステップ6] 応答をAG-001へ返す
```

### 2.4 設定・構成

#### 2.4.1 共通設定

| 設定項目 | 設定値 | 備考 |
|---------|--------|------|
| 会話管理 | SlidingWindowConversationManager(window_size=20, should_truncate_results=True, per_turn=False) | 複数ステップの情報収集・処理を行うため |
| リトライ戦略 | ModelRetryStrategy(max_attempts=6, initial_delay=4, max_delay=240) | 全エージェント共通 |
| フック | [HumanApprovalHook(target_tools=["generate_transport_form"]), LoopControlHook(max_iterations=10)] | 申請書生成前のユーザー承認必須（APR-001） |
| セッション管理 | FileSessionManager（session_idはinvocation_stateから取得） | 会話履歴の永続化 |
| callback_handler | None | ストリーミング無効化 |

#### 2.4.2 利用ツール

| ツールID | ツール名 | 利用目的 | 呼び出し条件 |
|----------|----------|----------|--------------|
| TL-001 | calculate_transport_fare | 移動情報から交通費を計算する | 移動情報収集完了後 |
| TL-002 | generate_transport_form | 交通費精算申請書を生成する | ユーザーのOK選択後（HumanApprovalHook通過後） |

### 2.5 連携設計

#### 2.5.1 Agent as Tools設計

**ツール名**: `transport_agent`

**ツール説明**:
```
交通費精算申請を処理する。移動情報を収集し、交通費を計算して申請書を生成する。
```

**ToolContextの使用**:
- `@tool(context=True)` デコレータを使用
- `tool_context.invocation_state` から `applicant_name`・`application_date`・`session_id` を取得

#### 2.5.2 invocation_stateの受け渡し

```python
# AG-001から受け取る
state = tool_context.invocation_state
session_id = state["session_id"]
applicant_name = state["applicant_name"]
application_date = state["application_date"]

# 子エージェントへはsession_idを除いて渡す
child_invocation_state = {
    "applicant_name": applicant_name,
    "application_date": application_date
}
response = agent(query, invocation_state=child_invocation_state)
```

## 3. 実装詳細

### 3.1 クラス/関数設計

#### 3.1.1 transport_agent 関数

##### 説明
AG-001からAgent as Toolsとして呼び出されるツール関数。invocation_stateから申請者情報を取得し、交通費精算申請エージェントを生成して処理を実行する。

##### 引数
- `query` (str): AG-001からの指示テキスト（必須）
- `tool_context` (ToolContext): ツールコンテキスト（フレームワーク注入）

##### 戻り値
- `str`: 処理結果テキスト（申請書生成完了メッセージまたはエラーメッセージ）

##### 処理内容
1. `invoke_specialist_agent(query, tool_context, agent_id="AG-002", deadline_months=3, build_agent=_build_transport_agent)` を呼び出す

#### 3.1.2 _build_transport_agent 関数

##### 説明
交通費精算申請エージェントのインスタンスを生成する。

##### 引数
- `session_id` (str): セッションID（必須）
- `applicant_name` (str): 申請者名（必須）
- `application_date` (str): 申請日（必須）
- `deadline` (str): 申請期限（必須）

##### 戻り値
- `Agent`: 交通費精算申請エージェントインスタンス

##### 処理内容
1. `get_transport_system_prompt(applicant_name, application_date, deadline)` でシステムプロンプトを生成
2. `create_specialist_agent(session_id, system_prompt, tools=[calculate_transport_fare, generate_transport_form], agent_name="交通費精算申請エージェント", ...)` でエージェントを生成

### 3.2 エラーハンドリング

| エラー種別 | 条件 | ログレベル | 対応 | 戻り値 |
|-----------|------|-----------|------|--------|
| LoopLimitError | LoopControlHookがループ上限超過を検知 | WARNING | ErrorHandler.handle_loop_limit_error()に委譲してstr返却 | str（エラーメッセージ） |
| ContextWindowOverflowException | コンテキストウィンドウ超過 | WARNING | ErrorHandler.handle_context_window_error()に委譲してstr返却 | str（エラーメッセージ） |
| MaxTokensReachedException | 最大トークン数到達 | WARNING | ErrorHandler.handle_max_tokens_error()に委譲してstr返却 | str（エラーメッセージ） |
| RuntimeError | その他の実行時エラー | ERROR（exc_info=True） | ErrorHandler.handle_runtime_error()に委譲してstr返却 | str（エラーメッセージ） |
| Exception | 予期しないエラー | ERROR（exc_info=True） | ErrorHandler.handle_unexpected_error()に委譲してstr返却 | str（エラーメッセージ） |

**注意**: 全例外のログにはクエリの先頭50文字を付加情報として含める（例: `query={query[:50]}`）

## 5. 補足情報

### 5.1 実装上の注意点

1. **base_agent.py の共通ユーティリティを使用**
   - `create_specialist_agent()` でエージェントを生成する

2. **HumanApprovalHook の登録**
   - `target_tools=["generate_transport_form"]` を設定する

3. **deadline_months の設定**
   - `settings.transport_agent.deadline_months = 3`（90日 = 3ヶ月）
