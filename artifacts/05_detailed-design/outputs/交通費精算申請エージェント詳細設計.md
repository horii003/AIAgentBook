## 1. 概要

### 1.1 コンポーネントの目的

交通費精算申請の移動情報を収集し、運賃を自動計算して交通費精算申請書（Excel）を生成する。AG-001 から Agent as Tool として呼び出される専門エージェント。

### 1.2 主要な責務

- 移動情報の収集（出発地・目的地・交通手段・移動日・業務目的）
- 交通費自動計算（TOOL-001 呼び出し）
- 申請期限チェック（経費発生日から3ヶ月以内）
- 上長承認要否判断（交通費合計10,000円超）
- 申請書ドラフト提示（テキスト形式、ツール呼び出しなし）
- 申請書生成（TOOL-002: generate_transport_expense_form 呼び出し）

### 2.2 Agent as Tools 設計

#### ツール関数

```python
@tool(context=True)
def transportation_expense_agent(query: str, tool_context: ToolContext) -> str:
    """交通費精算申請を処理する専門エージェント。
    移動情報の収集・交通費計算・申請書生成を担当する。

    Args:
        query (str): AG-001 からの指示・ユーザーの申請内容

    Returns:
        str: 処理結果メッセージ（申請書生成完了・エラーメッセージ等）
    """
    return invoke_specialist_agent(
        query=query,
        tool_context=tool_context,
        agent_id="AG-002",
        deadline_months=3,
        build_agent=_build_transportation_expense_agent,
    )
```

#### ビルド関数

```python
def _build_transportation_expense_agent(
    session_id: str,
    applicant_name: str,
    application_date: str,
    deadline: str,
) -> Agent:
    system_prompt = get_transportation_expense_system_prompt(
        applicant_name=applicant_name,
        application_date=application_date,
        deadline=deadline,
    )
    return create_specialist_agent(
        session_id=session_id,
        system_prompt=system_prompt,
        tools=[calculate_transport_fare, generate_transport_expense_form],
        agent_name="交通費精算申請エージェント",
        window_size=20,
        max_iterations=10,
        max_attempts=6,
        initial_delay=4,
        max_delay=240,
    )
```

## 3. ビジネスロジック

### 3.1 システムプロンプト

**生成方式**: 動的生成関数（`get_transportation_expense_system_prompt`）

**動的に埋め込む情報**:
- `applicant_name`: 申請者名
- `application_date`: 申請日（YYYY-MM-DD形式）
- `deadline`: 申請期限（application_date から3ヶ月前の日付）

**プロンプト全文**:
```
あなたは交通費精算申請エージェントです。申請者「{applicant_name}」の交通費精算申請を処理します。

## 基本情報
- 申請日: {application_date}
- 申請期限: {deadline}（経費発生日はこの日付以降であること）

## 役割
移動情報の収集・交通費計算・申請書生成を担当します。

## 処理フロー
1. 移動情報を一区間ずつ収集する（出発地・目的地・交通手段・移動日・業務目的）
2. 電車の場合は駅名を正規化してから calculate_transport_fare ツールを呼び出す
3. 申請期限チェック: 移動日が {deadline} 以降であることを確認する
4. 業務目的が記載されていることを確認する
5. 全区間の収集完了後、交通費合計を計算する
6. 交通費合計が10,000円を超える場合は上長承認要否フラグを True に設定する
7. 収集済み申請情報をテキストとして整理・提示する（申請書ドラフト提示）
8. ユーザーの承認（OK）後に generate_transport_expense_form ツールを呼び出す

## 業務ルール
- BRL-07: 移動情報は一区間ずつ収集する
- BRL-08: 運賃は calculate_transport_fare ツールで自動計算する
- BRL-09: 申請期限（{deadline}）を超過している場合は申請不可を案内して処理を終了する
- BRL-10: 交通費合計が10,000円を超える場合は上長承認欄を追加する
- BRL-11: 電車の場合は駅名を正規化してからツールに渡す
- BRL-12: 業務目的は必須。未記載時は入力を促す

## 禁止事項
- 申請書の提出は行わない（提出は人が実行する）
- ドラフト提示前に generate_transport_expense_form ツールを呼び出さない
- エラー発生時はエラー内容を要約して呼び出し元に返す
```

---

### 3.2 処理フロー

```
開始（AG-001 から委譲）
  ↓
移動情報収集（一区間ずつ）
  - 出発地・目的地・交通手段・移動日・業務目的を収集
  - 電車の場合は駅名を正規化
  - calculate_transport_fare ツールを呼び出して運賃を計算
  - 申請期限チェック（移動日 >= deadline）
    → 超過時: 申請不可を案内して終了
  - 業務目的確認（未記載時は入力を促す）
  ↓
全区間収集完了後
  - 交通費合計を計算
  - 10,000円超の場合は needs_approval=True に設定
  ↓
申請書ドラフト提示（テキスト形式、ツール呼び出しなし）
  ↓
ユーザーの承認（HumanApprovalHook: BeforeToolCallEvent でトリガー）
  → OK: generate_transport_expense_form ツールを呼び出す
  → 修正: 修正指示受付 → 再生成
  → キャンセル: 生成中止
  ↓
申請書ファイルパスを提示して終了
```

## 9. 補足情報

### 9.1 実装上の注意点

1. **業務ルール参照先**
   - 妥当性チェックルールは `knowledge/transportation_policies.py` を参照してプロンプトに組み込む（修正5）

2. **例外捕捉の順序**
   - `LoopLimitError` → `ContextWindowOverflowException` → `MaxTokensReachedException` → `RuntimeError` → `Exception` の順で捕捉する

3. **ログへの query 付加**
   - 全例外ログに `query` の先頭50文字を付加情報として含める

4. **エラー時の戻り値**
   - エラー時の戻り値はすべて `str`（呼び出し元 LLM コンテキストに渡されるため）
