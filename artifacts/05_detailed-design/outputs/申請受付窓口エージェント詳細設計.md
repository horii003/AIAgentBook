## 1. 概要

### 1.1 コンポーネントの目的

社員の申請内容を受け付け、申請種別（交通費精算申請・経費精算申請）を判断して申請先を提示する。申請種別確定後に専門エージェント（AG-002 または AG-003）へ処理を委譲する。

### 1.2 主要な責務

- 申請内容受付（ユーザーの自然文入力）
- 申請種別判断・提示（交通費精算申請 / 経費精算申請）
- 申請種別判断不能時の選択肢提示
- 専門エージェント（AG-002 / AG-003）への委譲
- 申請ルール案内
- セッション管理（セッションID生成・invocation_state 生成）

## 3. ビジネスロジック

### 3.1 システムプロンプト

**生成方式**: 静的定数（`ORCHESTRATOR_SYSTEM_PROMPT`）

**プロンプト全文**:
```
あなたは社内申請受付窓口エージェントです。社員の申請内容を受け付け、申請種別を判断して適切な専門エージェントに処理を委譲します。

## 役割
- 申請内容から申請種別（交通費精算申請・経費精算申請）を判断して提示する
- 申請種別確定後に専門エージェントへ処理を委譲する
- 申請ルール・方法が不明な利用者に社内申請ルールを案内する

## 申請種別の判断基準（BRL-01）
- 交通費精算申請: 申請内容に「交通費」「移動」「電車」「バス」「タクシー」「飛行機」「出張」「通勤」等のキーワードが含まれる場合
- 経費精算申請: 申請内容に「経費」「領収書」「購入」「宿泊」「資格」「備品」「事務用品」等のキーワードが含まれる場合

## 判断不能時の対応（BRL-02）
申請種別が判断できない場合は、以下の選択肢を提示してユーザーに確認する:
「申請種別を判断できませんでした。以下のどちらの申請ですか？
1. 交通費精算申請（電車・バス・タクシー・飛行機等の交通費）
2. 経費精算申請（備品購入・宿泊・資格取得等の経費）」

## 申請先の提示（BRL-03）
- 交通費精算申請 → transportation_expense_agent ツールを呼び出す
- 経費精算申請 → general_expense_agent ツールを呼び出す

## 社内申請ルール
- 申請期限: 経費発生日から3ヶ月以内
- 交通費10,000円超: 上長承認が必要
- 経費5,000円超: 上長承認が必要
- 申請書の提出は必ず人が実行する（AIは申請書を提出しない）

## 禁止事項
- 申請書の生成・提出は行わない（専門エージェントが担当）
- 申請種別の最終選択はユーザーが行う
```

### 3.2 処理フロー

```
開始
  ↓
ウェルカムメッセージを表示する
  ↓
申請者名を入力させる（空文字の場合は再入力を促す）
  ↓
セッションIDを生成する
  - 形式: {タイムスタンプ}_{UUID8文字}
  - 例: 20260323153045_a1b2c3d4
  - 生成方法: datetime.now().strftime("%Y%m%d%H%M%S") + "_" + uuid.uuid4().hex[:8]
  ↓
Agent() を初期化する（session_manager, hooks=[LoopControlHook] 等）
  ↓
対話ループ開始
  ↓
  input("\n\n入力内容（終了時はquit）: ") でユーザー入力を受け付ける
  ↓
  [exit / quit の場合]
    → "セッションを終了します。" を表示して break
  ↓
  [reset の場合]
    → 会話履歴・申請者情報をリセットして最初から開始する
  ↓
  [空文字の場合]
    → "入力内容を入力してください。" を表示して continue
  ↓
  [500文字超過の場合]
    → "入力内容は500文字以内で入力してください。" を表示して continue
  ↓
  invocation_state を生成する
    {
      "user_name": self._user_name,
      "request_date": datetime.now().strftime("%Y-%m-%d"),
      "session_id": self._session_id
    }
  ↓
  self.agent(user_input, invocation_state=invocation_state) を呼び出す
  ↓
  応答を print で表示する
  ↓
  [例外処理]
    → KeyboardInterrupt: INFO ログ → ErrorHandler.handle_keyboard_interrupt(e) → print → break
    → LoopLimitError: WARNING ログ → ErrorHandler.handle_loop_limit_error(e) → print → continue
    → ContextWindowOverflowException: WARNING ログ → ErrorHandler.handle_context_window_error(e) → print → continue
    → MaxTokensReachedException: WARNING ログ → ErrorHandler.handle_max_tokens_error(e) → print → continue
    → RuntimeError: ERROR ログ（exc_info=True） → ErrorHandler.handle_runtime_error(e) → print → continue
    → Exception: ERROR ログ（exc_info=True） → ErrorHandler.handle_unexpected_error(e) → print → continue
  ↓
対話ループ継続
```

### 3.3 振り分けロジック

| キーワード | 振り分け先 | ツール名 |
|-----------|-----------|---------|
| 交通費・移動・電車・バス・タクシー・飛行機・出張・通勤 | AG-002 | `transportation_expense_agent` |
| 経費・領収書・購入・宿泊・資格・備品・事務用品 | AG-003 | `general_expense_agent` |
| 判断不能 | ユーザーに選択肢を提示 | — |

## 10. 補足情報

### 10.1 実装上の注意点

1. **セッションID 形式**
   - `{タイムスタンプ（秒単位）}_{UUID（8文字）}` 形式
   - 例: `20260323153045_a1b2c3d4`
   - `datetime.now().strftime("%Y%m%d%H%M%S") + "_" + uuid.uuid4().hex[:8]` で生成

2. **例外捕捉の順序**
   - `KeyboardInterrupt` → `LoopLimitError` → `ContextWindowOverflowException` → `MaxTokensReachedException` → `RuntimeError` → `Exception` の順で捕捉する

3. **ログへの session_id 付加**
   - 全例外ログに `session_id` を付加情報として含める

## 12. テスト観点

### 12.1 機能テスト
- 申請種別が交通費精算申請として正しく判断され、`transportation_expense_agent` が呼び出されること
- 申請種別が経費精算申請として正しく判断され、`general_expense_agent` が呼び出されること

- 空文字入力で再入力が促されること
- 500文字超過入力で再入力が促されること
- LoopLimitError 発生時にエラーメッセージが表示されてループが継続すること
- ContextWindowOverflowException 発生時にエラーメッセージが表示されてループが継続すること
- MaxTokensReachedException 発生時にエラーメッセージが表示されてループが継続すること

- KeyboardInterrupt 発生時にエラーメッセージが表示されてループが終了すること
