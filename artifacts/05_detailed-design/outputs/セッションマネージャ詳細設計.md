## 1. 概要

### 1.1 コンポーネントの目的

Strands SDKの `FileSessionManager` を薄くラップし、セッションIDとストレージパスを統一的に管理する。各エージェント（AG-001〜AG-003）が独立したセッションを持ち、会話履歴をローカルファイルに永続化する。セッションIDの生成規則を一元管理し、エージェント間でのセッション識別を保証する。

### 1.2 主要な責務

- **セッションID生成**: `{YYYYMMDD_HHMMSS}_{8桁ランダム英数字}` 形式のセッションIDを生成する
- **FileSessionManager生成**: session_idとstorage_dirを指定してFileSessionManagerインスタンスを生成する
- **ストレージパス管理**: セッションデータの保存先（`storage/sessions`）を統一管理する

## 4. セッションライフサイクル設計

### 4.1 セッション生成（main.py起動時）

```
main.py 起動
  ↓
SessionManagerFactory.generate_session_id() でsession_idを生成
  - 形式: {YYYYMMDD_HHMMSS}_{8桁ランダム英数字}
  ↓
SessionManagerFactory.create_session_manager(session_id) でAG-001用FileSessionManagerを生成
  ↓
AG-001のAgent()コンストラクタにsession_managerを渡す
  ↓
session_idをinvocation_stateに含めてAG-001のagent()呼び出し時に渡す
```

### 4.2 専門エージェントのセッション生成（AG-002/AG-003起動時）

```
AG-002/AG-003のAgent as Toolsラッパー関数が呼び出される
  ↓
tool_context.invocation_state.get("session_id") でsession_idを取得
  ↓
SessionManagerFactory.create_session_manager(session_id) でFileSessionManagerを生成
  ↓
AG-002/AG-003のAgent()コンストラクタにsession_managerを渡す
  ↓
子エージェントへのinvocation_stateにはsession_idを含めない（ファクトリ関数で消費済み）
```

### 4.3 セッション復元

```
FileSessionManager生成時（session_idを指定）
  ↓
storage/sessions/{session_id}/ にセッションファイルが存在するか確認
  ↓
存在する → FileSessionManagerが自動的に会話履歴を復元する
存在しない → 新規セッションとして開始する
```

### 4.4 セッション終了

```
申請書生成完了またはユーザーがキャンセルを選択
  ↓
セッション状態をCLOSEDに遷移（エージェントが管理）
  ↓
セッションファイルはstorage/sessions/に保持される（削除は要件上未定義）
```

## 5. 永続化設計

### 5.1 ファイルパス生成ルール

| 項目 | 値 |

```

└── sessions/

    ├── {session_id}/    # AG-002のセッションデータ（同一session_idを使用）
    └── {session_id}/    # AG-003のセッションデータ（同一session_idを使用）
```

## 6. Strands SDK連携設計

### 6.1 ConversationManager設定

| エージェント種別 | クラス | storage_dir | window_size |
|--------------|-------|------------|------------|
| オーケストレーター（AG-001） | `FileSessionManager` | `storage/sessions` | 30 |
| 専門エージェント（AG-002/AG-003） | `FileSessionManager` | `storage/sessions` | 20 |

### 6.2 エージェントへの渡し方

```python
from strands.session import FileSessionManager
from session.session_manager import SessionManagerFactory

# AG-001用（main.pyで生成）
session_id = SessionManagerFactory.generate_session_id()
session_manager = SessionManagerFactory.create_session_manager(session_id)

agent = Agent(
    ...,
    session_manager=session_manager,
    conversation_manager=SlidingWindowConversationManager(
        window_size=30,
        should_truncate_results=True,
        per_turn=False
    )
)

# AG-002/AG-003用（Agent as Toolsラッパー関数内で生成）
session_id = tool_context.invocation_state.get("session_id")
session_manager = SessionManagerFactory.create_session_manager(session_id)

agent = Agent(
    ...,
    session_manager=session_manager,
    conversation_manager=SlidingWindowConversationManager(
        window_size=20,
        should_truncate_results=True,
        per_turn=False
    )
)
```

## 7. 主要メソッド設計

### 7.1 generate_session_id

| 項目 | 値 |
|------|-----|
| 説明 | `{YYYYMMDD_HHMMSS}_{8桁ランダム英数字}` 形式のセッションIDを生成する |
| 引数 | なし |
| 戻り値 | `str`（セッションID） |
| デコレータ | `@staticmethod` |

**処理フロー:**
```
datetime.now().strftime("%Y%m%d_%H%M%S") でタイムスタンプを生成
  ↓
secrets.token_hex(4) で8桁ランダム英数字を生成
  ↓
"{timestamp}_{random}" 形式で結合して返す
```

**戻り値の例**: `"20260510_114355_a1b2c3d4"`

---

### 7.2 create_session_manager

| 項目 | 値 |
|------|-----|
| 説明 | 指定したsession_idとstorage_dirでFileSessionManagerインスタンスを生成する |
| 引数 | `session_id: str`、`storage_dir: str = DEFAULT_STORAGE_DIR` |
| 戻り値 | `FileSessionManager` |
| デコレータ | `@classmethod` |

**処理フロー:**
```
FileSessionManager(session_id=session_id, storage_dir=storage_dir) を生成
  ↓
FileSessionManagerインスタンスを返す
```

**例外:**

| 例外クラス | 発生条件 |
|----------|---------|
| `OSError` | storage_dirへの書き込み権限がない場合 |

---

### 7.3 get_session_storage_path

| 項目 | 値 |
|------|-----|
| 説明 | セッションデータの保存先パスを返す |
| 引数 | `session_id: str`、`storage_dir: str = DEFAULT_STORAGE_DIR` |
| 戻り値 | `str`（セッションデータの保存先パス） |
| デコレータ | `@staticmethod` |

**処理フロー:**
```
os.path.join(storage_dir, session_id) でパスを生成して返す
```

## 8. エラーハンドリング

### 8.1 エラー処理フロー

```
OSError 発生（storage_dirへの書き込み権限なし等）
  ↓
呼び出し元（main.py または Agent as Toolsラッパー関数）がキャッチ
  ↓
_logger.error("セッションマネージャの生成に失敗しました", exc_info=True)
  ↓
ErrorHandler.handle_unexpected_error(e) に委譲
  ↓
エラーメッセージをユーザーへ表示して処理中断
```

### 8.2 エラーメッセージ

| エラー種別 | 発生条件 | ユーザー向けメッセージ |
|----------|---------|---------------------|
| OSError | storage_dirへの書き込み権限なし | `"申し訳ありません。データの読み込みに失敗しました。システム管理者にご連絡ください。"` |

## 10. 補足情報

### 10.1 実装上の注意点

1. **薄いラッパー設計**
   - FileSessionManagerの機能をそのまま利用し、セッションID生成とパス管理のみを追加する
   - 会話履歴の管理はFileSessionManagerに委譲する

2. **session_idの伝播**
   - AG-001が生成したsession_idをinvocation_state経由でAG-002/AG-003へ伝播する
   - 子エージェントへのinvocation_stateにはsession_idを含めない（ファクトリ関数で消費済み）

3. **ログ出力の有無**
   - `SessionManagerFactory` はログ出力を行う（`_logger = logging.getLogger(__name__)` を定義する）

## 12. テスト観点

### 12.1 セッションライフサイクルテスト
- `generate_session_id()` が `{YYYYMMDD_HHMMSS}_{8桁英数字}` 形式のIDを生成すること
- `generate_session_id()` が呼び出しごとに異なるIDを生成すること
- `create_session_manager()` が `FileSessionManager` インスタンスを返すこと

### 12.2 永続化テスト
- 既存セッションファイルが存在する場合に会話履歴が復元されること
- `storage/sessions` ディレクトリが存在しない場合に自動作成されること

### 12.3 異常系テスト
- `storage/sessions` への書き込み権限がない場合に `OSError` が発生すること

### 12.4 境界値テスト
- セッションIDが正確に `{YYYYMMDD_HHMMSS}_{8桁英数字}` 形式であること（長さ・文字種の確認）
