## 1. 概要

### 1.1 コンポーネントの目的

Strands SDK の `FileSessionManager` を統一的に生成・管理するファクトリクラスを提供する。セッションIDの生成ロジックを一元化し、全エージェント（AG-001, AG-002, AG-003）で一貫したセッション管理を実現する。セッションの再開（Resume）に対応し、中断後の会話継続を可能にする。

### 1.2 主要な責務

1. **セッションID生成**: `{YYYYMMDDHHMMSS}_{UUID8文字}` 形式のセッションIDを生成する。生成タイミングはセッション開始時にAG-001が生成する
2. **FileSessionManager 生成**: セッションIDと保存先パスを指定して `FileSessionManager` インスタンスを生成する
3. **保存先ディレクトリ管理**: `storage/sessions/` ディレクトリの存在確認・自動作成を行う
4. **セッション存在確認**: 指定されたセッションIDのセッションデータが存在するかを確認する

## 2. クラス基本情報

| 項目 | 値 |
|------|-----|
| クラス名 | `SessionManagerFactory` |
| ファイルパス | `session/session_manager.py` |
| 説明 | Strands SDK の FileSessionManager インスタンスを生成するファクトリクラス。セッションIDの生成と保存先ディレクトリの管理を担当する。全メソッドが `@staticmethod` であり、インスタンス化不要。 |

## 3. 初期化設計

### 3.1 コンストラクタ

`SessionManagerFactory` はインスタンス変数を持たない。`@staticmethod` のみで構成されるため、初期化不要。

**クラス変数**:

| 変数名 | 型 | 初期値 | 説明 |
|--------|-----|--------|------|
| `DEFAULT_STORAGE_DIR` | `str` | `"storage/sessions"` | セッションデータの保存先ディレクトリパス |

## 4. セッションライフサイクル設計

### 4.1 セッション生成

```
main.py 起動
  ↓
AG-001（OrchestratorAgent）初期化時にセッションIDを生成する
  → session_id = datetime.now().strftime("%Y%m%d%H%M%S") + "_" + uuid.uuid4().hex[:8]
  → 例: "20260323153045_a1b2c3d4"
  ↓
SessionManagerFactory.create_session_manager(session_id) を呼び出す
  → storage/sessions/ ディレクトリが存在しない場合は os.makedirs(exist_ok=True) で作成
  → FileSessionManager(session_id=session_id, storage_dir=DEFAULT_STORAGE_DIR) を生成
  → FileSessionManager インスタンスを返す
  ↓
AG-001 の Agent() コンストラクタの session_manager パラメータに渡す
```

### 4.2 セッション保存

```
エージェントが応答を返す（会話ターン完了）
  ↓
FileSessionManager が自動的に会話履歴を storage/sessions/{session_id}/ に保存する
  （Strands SDK の内部処理。SessionManagerFactory は関与しない）
```

### 4.3 セッション復元

```
main.py 起動時にユーザーが既存セッションIDを入力
  ↓
SessionManagerFactory.session_exists(session_id) を呼び出す
  → storage/sessions/{session_id}/ ディレクトリが存在するか確認
  ↓
存在する → SessionManagerFactory.create_session_manager(session_id) を呼び出す
           → FileSessionManager が既存セッションデータを自動復元
           → 会話履歴が復元された FileSessionManager インスタンスを返す
存在しない → AG-001が新規セッションIDを生成して新規セッション生成フローへ
```

### 4.4 セッション削除

```
（明示的な削除は本システムでは実装しない。
  セッションデータは storage/sessions/ に永続化され、
  保持期間は運用設計フェーズで決定する）
```

## 5. 永続化設計

### 5.1 ファイルパス生成ルール

| 項目 | 値 |
|------|-----|
| 基底ディレクトリ | `storage/sessions/` |
| セッションディレクトリ | `storage/sessions/{session_id}/` |
| ファイル形式 | FileSessionManager の内部実装に依存（JSON 形式） |
| エンコード | UTF-8 |

### 5.2 永続化データ構造

FileSessionManager が内部的に管理するため、SessionManagerFactory は直接操作しない。保存されるデータは会話履歴（メッセージリスト）。

## 6. Strands SDK 連携設計

### 6.1 ConversationManager 設定

| エージェント種別 | エージェント | window_size | should_truncate_results | per_turn |
|--------------|------------|------------|------------------------|---------|
| オーケストレーター | AG-001 | 30 | True | False |
| 専門エージェント（複雑） | AG-003 | 20 | True | False |
| 専門エージェント（単純） | AG-002 | 15 | True | False |

### 6.2 エージェントへの渡し方

```python
from strands import Agent, ModelRetryStrategy
from strands.agent.conversation_manager import SlidingWindowConversationManager
from session.session_manager import SessionManagerFactory

# セッション生成（AG-001 OrchestratorAgent 初期化時）
import uuid
from datetime import datetime
session_id = datetime.now().strftime("%Y%m%d%H%M%S") + "_" + uuid.uuid4().hex[:8]
session_manager = SessionManagerFactory.create_session_manager(session_id)

# AG-001（オーケストレーター）への渡し方
agent = Agent(
    ...
    conversation_manager=SlidingWindowConversationManager(
        window_size=30,
        should_truncate_results=True,
        per_turn=False
    ),
    session_manager=session_manager,
    ...
)
```

## 7. 主要メソッド設計

### 7.1 generate_session_id

| 項目 | 値 |
|------|-----|
| 説明 | 一意なセッションIDを生成する。生成タイミング: セッション開始時にAG-001が生成する |
| 引数 | なし |
| 戻り値 | `str`（セッションID） |

**処理フロー**:
```
datetime.now().strftime("%Y%m%d%H%M%S") でタイムスタンプ部分を生成
  ↓
uuid.uuid4().hex[:8] で8文字のUUID文字列を生成
  ↓
f"{timestamp}_{uuid_str}" を返す
```

**例**: `"20260323153045_a1b2c3d4"`

---

### 7.2 create_session_manager

| 項目 | 値 |
|------|-----|
| 説明 | 指定されたセッションIDで FileSessionManager インスタンスを生成する |
| 引数 | `session_id: str`（セッションID） |
| 戻り値 | `FileSessionManager` |

**処理フロー**:
```
storage_dir = DEFAULT_STORAGE_DIR
  ↓
os.makedirs(storage_dir, exist_ok=True) でディレクトリを作成（存在する場合はスキップ）
  ↓
FileSessionManager(session_id=session_id, storage_dir=storage_dir) を生成
  ↓
FileSessionManager インスタンスを返す
```

**例外**:

| 例外クラス | 発生条件 |
|----------|---------|
| `OSError` | ディレクトリ作成権限がない場合 |

---

### 7.3 session_exists

| 項目 | 値 |
|------|-----|
| 説明 | 指定されたセッションIDのセッションデータが存在するかを確認する |
| 引数 | `session_id: str`（セッションID） |
| 戻り値 | `bool`（存在する場合 True） |

**処理フロー**:
```
session_dir = get_session_dir(session_id)
  ↓
os.path.exists(session_dir) を確認
  ↓
存在する → True を返す
存在しない → False を返す
```

---

### 7.4 get_session_dir

| 項目 | 値 |
|------|-----|
| 説明 | 指定されたセッションIDに対応するセッションデータの保存ディレクトリパスを返す |
| 引数 | `session_id: str`（セッションID） |
| 戻り値 | `str`（セッションデータの保存ディレクトリパス） |

**処理フロー**:
```
f"{DEFAULT_STORAGE_DIR}/{session_id}" を返す
```

**例**: `"storage/sessions/20260323153045_a1b2c3d4"`

## 8. エラーハンドリング

### 8.1 エラー処理フロー

```
エラー発生（create_session_manager 内）
  ↓
OSError（ディレクトリ作成失敗）
  → _logger.error("セッションディレクトリ作成エラー: %s", e, exc_info=True)
  → 例外を再送出する（呼び出し元が処理）
```

### 8.2 エラーメッセージ

| エラー種別 | 発生条件 | ユーザー向けメッセージ |
|----------|---------|---------------------|
| OSError（EX-07） | storage/sessions/ ディレクトリ作成権限がない | 呼び出し元（main.py）が ErrorHandler.handle_unexpected_error() で生成 |

## 10. 補足情報

### 10.1 実装上の注意点

1. **全メソッド @staticmethod**
   - インスタンス化不要。`SessionManagerFactory.generate_session_id()` として直接呼び出す

2. **uuid モジュールの使用**
   - ランダム文字列生成には `uuid.uuid4().hex[:8]` を使用する
   - 生成タイミング: セッション開始時にAG-001が生成する（SessionManagerFactoryではなくAG-001が生成）

3. **ログを出力するモジュールのみ _logger を定義**
   - `session_manager.py` はログ出力を行うため、モジュール先頭に `_logger = logging.getLogger(__name__)` を定義する

## 12. テスト観点

### 12.1 セッションライフサイクルテスト
- `generate_session_id` が `{YYYYMMDDHHMMSS}_{UUID8文字}` 形式のIDを返すこと
- `create_session_manager` が FileSessionManager インスタンスを返すこと
- `create_session_manager` 呼び出し時に `storage/sessions/` ディレクトリが自動作成されること
- `session_exists` が既存セッションに対して True を返すこと
- `session_exists` が存在しないセッションに対して False を返すこと

### 12.2 永続化テスト
- 同一セッションIDで `create_session_manager` を2回呼び出した場合に既存セッションが復元されること

### 12.3 異常系テスト
- `storage/sessions/` ディレクトリへの書き込み権限がない場合に OSError が発生すること

### 12.4 境界値テスト
- 100回連続で `generate_session_id` を呼び出した場合に全て一意のIDが生成されること
