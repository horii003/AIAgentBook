## 1. 概要

### 1.1 コンポーネントの目的

AWS Strands SDK の FileSessionManager をラップし、セッションID生成とセッションデータの永続化を担当する。

### 1.2 主要な責務

- セッションID の生成（`{タイムスタンプ}_{UUID8文字}` 形式）
- FileSessionManager インスタンスの生成
- セッションデータの永続化（`storage/sessions/` ディレクトリ）

---

## 2. 設計詳細

### 2.1 クラス基本情報

#### クラス名
`SessionManager`

#### 説明
FileSessionManager の薄いラッパークラス。セッションID生成と FileSessionManager インスタンス生成のみを担当する。

---

### 2.2 初期化

#### `__init__(self)`

**インスタンス変数**:
- `_session_id`: `str` — 現在のセッション識別子（初期値: `""`）
- `_session_manager`: `FileSessionManager | None` — FileSessionManager インスタンス（初期値: `None`）
- `_storage_dir`: `str` — セッションデータの保存先ディレクトリ（初期値: `"storage/sessions"`）

---

### 2.3 主要メソッド

#### 2.3.1 generate_session_id

##### 説明
`{タイムスタンプ（秒単位）}_{UUID（8文字）}` 形式のセッション識別子を生成する。

##### 引数
なし

##### 戻り値
- `str`: セッション識別子

##### 処理内容
1. `datetime.now().strftime("%Y%m%d%H%M%S")` でタイムスタンプを生成する
2. `uuid.uuid4().hex[:8]` で UUID の先頭8文字を生成する
3. `{タイムスタンプ}_{UUID8文字}` 形式で結合して返す

##### セッションID 形式
- 形式: `{YYYYMMDDHHMMSS}_{UUID8文字}`
- 例: `20260323153045_a1b2c3d4`
- 生成方法: `datetime.now().strftime("%Y%m%d%H%M%S") + "_" + uuid.uuid4().hex[:8]`
- 生成タイミング: セッション開始時に AG-001 が生成する

---

#### 2.3.2 create_session_manager

##### 説明
セッションIDを使用して FileSessionManager インスタンスを生成する。

##### 引数
- `session_id` (`str`): セッション識別子

##### 戻り値
- `FileSessionManager`: AWS Strands SDK 提供のセッション管理クラス

##### 処理内容
1. `session_dir = f"{self._storage_dir}/{session_id}"` でセッションディレクトリパスを生成する
2. `FileSessionManager(session_id=session_id, storage_dir=session_dir)` でインスタンスを生成する
3. `self._session_id = session_id` に設定する
4. `self._session_manager = session_manager` に設定する
5. `_logger.info("セッション作成: session_id=%s", session_id)` でログ出力する
6. `session_manager` を返す

---

#### 2.3.3 get_session_id

##### 説明
現在のセッション識別子を返す。

##### 引数
なし

##### 戻り値
- `str`: セッション識別子

---

#### 2.3.4 get_session_manager

##### 説明
現在の FileSessionManager インスタンスを返す。

##### 引数
なし

##### 戻り値
- `FileSessionManager | None`: FileSessionManager インスタンス

---

## 3. ビジネスロジック

### 3.1 セッション生成フロー

```
main.py 起動時
  ↓
SessionManager() を初期化する
  ↓
session_id = session_manager.generate_session_id()
  - 形式: {YYYYMMDDHHMMSS}_{UUID8文字}
  - 例: 20260323153045_a1b2c3d4
  ↓
file_session_manager = session_manager.create_session_manager(session_id)
  - storage/sessions/{session_id}/ ディレクトリを使用
  ↓
AG-001 の Agent() コンストラクタに file_session_manager を渡す
  ↓
AG-001 が invocation_state に session_id を設定して AG-002/AG-003 に伝播する
```

### 3.2 専門エージェントでのセッション利用

```
AG-002/AG-003 のツール関数内
  ↓
tool_context.invocation_state から session_id を取得する
  ↓
ファクトリ関数内で FileSessionManager を生成する
  - FileSessionManager(session_id=session_id, storage_dir=f"storage/sessions/{session_id}")
  ↓
Agent() コンストラクタに FileSessionManager を渡す
```

## 6. 使用例

### 6.1 基本的な使用方法

```python
from session.session_manager import SessionManager

# main.py での使用例
session_manager = SessionManager()
session_id = session_manager.generate_session_id()
# 例: "20260323153045_a1b2c3d4"

file_session_manager = session_manager.create_session_manager(session_id)

# AG-001 の Agent() コンストラクタに渡す
agent = Agent(
    ...,
    session_manager=file_session_manager
)
```

## 7. 補足情報

### 7.1 実装上の注意点

1. **セッションID 形式**
   - `{YYYYMMDDHHMMSS}_{UUID8文字}` 形式（例: `20260323153045_a1b2c3d4`）
   - `datetime.now().strftime("%Y%m%d%H%M%S") + "_" + uuid.uuid4().hex[:8]` で生成する
   - 生成タイミング: セッション開始時に AG-001 が生成する

2. **セッションデータ保存先**
   - `storage/sessions/{session_id}/` ディレクトリに保存する
   - ディレクトリは FileSessionManager が自動作成する

## 9. テスト観点

### 9.1 機能テスト
- `generate_session_id()` が `{YYYYMMDDHHMMSS}_{UUID8文字}` 形式の文字列を返すこと
- `create_session_manager(session_id)` が FileSessionManager インスタンスを返すこと
- 同一タイムスタンプでも異なるランダム文字列により一意性が保証されること

### 9.2 性能テスト
- 複数セッションが同時に存在する場合にセッションデータが混在しないこと
