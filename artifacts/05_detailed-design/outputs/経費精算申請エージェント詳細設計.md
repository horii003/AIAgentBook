## 1. 概要

### 1.1 コンポーネントの目的

経費精算申請の経費情報を収集し、経費精算申請書（Excel）を生成する。AG-001 から Agent as Tool として呼び出される専門エージェント。領収書画像からの情報抽出・経費区分判断・申請書生成を担当する。

### 1.2 主要な責務

- 領収書画像からの経費情報抽出（LLM 画像解析機能）
- 経費区分の自動判断（LLM プロンプトベース）
- 申請期限チェック（経費発生日から3ヶ月以内）
- 上長承認要否判断（経費合計5,000円超）
- 申請書ドラフト提示（テキスト形式、ツール呼び出しなし）
- 申請書生成（TOOL-002: generate_general_expense_form 呼び出し）

### 2.2 Agent as Tools 設計

#### ツール関数

```python
@tool(context=True)
def general_expense_agent(query: str, tool_context: ToolContext) -> str:
    """経費精算申請を処理する専門エージェント。
    領収書情報収集・経費区分判断・申請書生成を担当する。

    Args:
        query (str): AG-001 からの指示・ユーザーの申請内容

    Returns:
        str: 処理結果メッセージ（申請書生成完了・エラーメッセージ等）
    """
    return invoke_specialist_agent(
        query=query,
        tool_context=tool_context,
        agent_id="AG-003",
        deadline_months=3,
        build_agent=_build_general_expense_agent,
    )
```

#### ビルド関数

```python
def _build_general_expense_agent(
    session_id: str,
    applicant_name: str,
    application_date: str,
    deadline: str,
) -> Agent:
    system_prompt = get_general_expense_system_prompt(
        applicant_name=applicant_name,
        application_date=application_date,
        deadline=deadline,
    )
    return create_specialist_agent(
        session_id=session_id,
        system_prompt=system_prompt,
        tools=[generate_general_expense_form],
        agent_name="経費精算申請エージェント",
        window_size=15,
        max_iterations=10,
        max_attempts=6,
        initial_delay=4,
        max_delay=240,
    )
```

## 3. ビジネスロジック

### 3.1 システムプロンプト

**生成方式**: 動的生成関数（`get_general_expense_system_prompt`）

**動的に埋め込む情報**:
- `applicant_name`: 申請者名
- `application_date`: 申請日（YYYY-MM-DD形式）
- `deadline`: 申請期限（application_date から3ヶ月前の日付）

**プロンプト全文**:
```
あなたは経費精算申請エージェントです。申請者「{applicant_name}」の経費精算申請を処理します。

## 基本情報
- 申請日: {application_date}
- 申請期限: {deadline}（経費発生日はこの日付以降であること）

## 役割
領収書情報の収集・経費区分判断・申請書生成を担当します。

## 処理フロー
1. 領収書画像または手動入力で経費情報を収集する（購入日・店舗名・品目・金額・業務目的）
2. 品目から経費区分を自動判断する（事務用品費・宿泊費・資格精算費・その他経費）
3. 申請期限チェック: 購入日が {deadline} 以降であることを確認する
4. 業務目的が記載されていることを確認する
5. 全経費の収集完了後、経費合計を計算する
6. 経費合計が5,000円を超える場合は上長承認要否フラグを True に設定する
7. 収集済み申請情報をテキストとして整理・提示する（申請書ドラフト提示）
8. ユーザーの承認（OK）後に generate_general_expense_form ツールを呼び出す

## 経費区分の判断基準（BRL-14）
- 事務用品費: 文具・コピー用紙・トナー等の事務用品
- 宿泊費: 出張時のホテル・旅館等の宿泊費
- 資格精算費: 資格取得・試験・研修等の費用
- その他経費: 上記に該当しない経費

## 業務ルール
- BRL-12: 業務目的は必須。未記載時は入力を促す
- BRL-13: 領収書画像から購入日・店舗名・品目・金額を自動抽出する。失敗時は手動入力を促す
- BRL-14: 品目から経費区分を自動判断する。判断不能時はユーザーに選択肢を提示する
- BRL-15: 申請期限（{deadline}）を超過している場合は申請不可を案内して処理を終了する
- BRL-16: 経費合計が5,000円を超える場合は上長承認欄を追加する

## 禁止事項
- 申請書の提出は行わない（提出は人が実行する）
- ドラフト提示前に generate_general_expense_form ツールを呼び出さない
- エラー発生時はエラー内容を要約して呼び出し元に返す
```

---

### 3.2 処理フロー

```
開始（AG-001 から委譲）
  ↓
領収書画像受付（または手動入力）
  - 画像解析（LLM の画像解析機能）
  - 経費情報抽出（購入日・店舗名・品目・金額）
  - 読み取り失敗時: 手動入力を促す
  ↓
経費区分判断（LLM プロンプトベース）
  - 判断不能時: ユーザーに選択肢を提示
  ↓
申請期限チェック（購入日 >= deadline）
  → 超過時: 申請不可を案内して終了
  ↓
業務目的確認（未記載時は入力を促す）
  ↓
全経費収集完了後
  - 経費合計を計算
  - 5,000円超の場合は needs_approval=True に設定
  ↓
申請書ドラフト提示（テキスト形式、ツール呼び出しなし）
  ↓
ユーザーの承認（HumanApprovalHook: BeforeToolCallEvent でトリガー）
  → OK: generate_general_expense_form ツールを呼び出す
  → 修正: 修正指示受付 → 再生成
  → キャンセル: 生成中止
  ↓
申請書ファイルパスを提示して終了
```

## 9. 補足情報

### 9.1 実装上の注意点

1. **業務ルール参照先**
   - 妥当性チェックルールは `knowledge/receipt_policies.py` を参照してプロンプトに組み込む（修正5）

2. **例外捕捉の順序**
   - `LoopLimitError` → `ContextWindowOverflowException` → `MaxTokensReachedException` → `RuntimeError` → `Exception` の順で捕捉する

3. **ログへの query 付加**
   - 全例外ログに `query` の先頭50文字を付加情報として含める

4. **エラー時の戻り値**
   - エラー時の戻り値はすべて `str`（呼び出し元 LLM コンテキストに渡されるため）
