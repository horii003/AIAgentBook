## 1. 概要

### 1.1 コンポーネントの目的

収集した申請情報を申請書テンプレート（Excel）に入力し、申請書ファイルを生成・保存する。エージェント（AG-002, AG-003）が対話で収集した情報を正確に Excel ファイルに反映し、生成した申請書ファイルのパスをエージェントに返す。

### 1.2 主要な責務

1. **invocation_state 参照**: `tool_context.invocation_state` から申請者名・申請日を取得する
2. **入力バリデーション**: `ExpenseReportInput` / `TransportReportInput` Pydantic モデルで入力パラメータを検証する
3. **テンプレート読み込み**: `template/` 配下の申請書テンプレート（.xlsx）を読み込む
4. **セル書き込み**: 申請者名・申請日・明細情報・合計金額をテンプレートの所定セルに書き込む
5. **ファイル保存**: `output/` 配下にタイムスタンプ付きファイル名で保存する
6. **結果返却**: 生成成否・ファイルパス・エラーメッセージを辞書形式で返却する

## 3. generate_expense_report ツール設計

### 3.1 基本情報

#### ツール名（関数名）
`generate_expense_report`

#### 説明（docstring）
```
経費精算申請書（Excel）を生成して output/ に保存する。

申請者名・申請日は invocation_state から自動取得する。
HumanApprovalHook による承認（OK選択）後にのみ実行される。

Args:
    items (list[dict]): 経費明細のリスト。各要素は以下のキーを持つ辞書:
        - purchase_date (str): 購入日（YYYY-MM-DD 形式）【必須】
        - store_name (str): 店舗名【必須】
        - item_name (str): 品目【必須】
        - expense_category (str): 経費区分。以下のいずれか【必須】
            "事務用品費", "宿泊費", "資格精算費", "その他経費"
        - amount (int): 金額（円、0以上）【必須】
        - business_purpose (str): 業務目的【必須】

Returns:
    dict: 以下のキーを持つ辞書:
        - success (bool): 生成成否
        - file_path (str | None): 生成ファイルパス。成功時のみ設定
        - error_message (str | None): エラーメッセージ。失敗時のみ設定
```

---

### 3.2 インターフェース設計

#### 3.2.1 入力

**パラメータ定義**:

| 引数名 | 型 | 説明 | デフォルト値 |
|--------|-----|------|--------------|
| `items` | `list[dict]` | 経費明細リスト（1件以上必須） | なし（必須） |
| `tool_context` | `ToolContext` | ツールコンテキスト（invocation_state を含む） | なし（フレームワーク注入） |

**items 各要素のフィールド**:

| フィールド名 | 型 | 必須/任意 | 説明 | 制約 |
|------------|-----|-----------|------|------|
| `purchase_date` | `str` | 必須 | 購入日 | YYYY-MM-DD 形式、validate_date_format 適用 |
| `store_name` | `str` | 必須 | 店舗名 | min_length=1 |
| `item_name` | `str` | 必須 | 品目 | min_length=1 |
| `expense_category` | `str` | 必須 | 経費区分 | `Literal["事務用品費","宿泊費","資格精算費","その他経費"]` |
| `amount` | `int` | 必須 | 金額（円） | ge=0 |
| `business_purpose` | `str` | 必須 | 業務目的 | min_length=1 |

**invocation_state 経由の入力**:

| キー名 | 型 | 説明 | 不足時の対応 |
|--------|-----|------|------------|
| `applicant_name` | `str` | 申請者名 | エラーメッセージを返して終了 |
| `application_date` | `str` | 申請日（YYYY-MM-DD） | エラーメッセージを返して終了 |

##### バリデーションルール詳細

| フィールド | ルール | エラーメッセージ |
|-----------|--------|----------------|
| `items` | min_length=1（空リスト禁止） | "経費明細が入力されていません。" |
| `purchase_date` | YYYY-MM-DD 形式 | "購入日のエラー: 日付の形式が正しくありません（YYYY-MM-DD 形式で入力してください）。" |
| `expense_category` | 許容値チェック | "経費区分のエラー: 事務用品費・宿泊費・資格精算費・その他経費のいずれかを指定してください。" |
| `amount` | ge=0 | "金額のエラー: 0以上の整数を入力してください。" |
| `store_name` / `item_name` / `business_purpose` | min_length=1 | "{フィールド名}のエラー: 入力してください。" |

#### 3.2.2 出力

**戻り値の型**: `dict`

**構造**:
```python
{
    "success": bool,              # 生成成否
    "file_path": str | None,      # 生成ファイルパス。成功時のみ設定
    "error_message": str | None   # エラーメッセージ。失敗時のみ設定
}
```

#### 3.2.3 例外

| 例外 | 発生条件 |
|------|---------|
| 発生しない | ツール関数は例外を再送出せず、辞書形式でエラーを返す |

### 3.3 ビジネスロジック

#### 3.3.1 処理フロー

```
開始
  ↓
[ステップ1] invocation_state から申請者名・申請日を取得
  - applicant_name = tool_context.invocation_state.get("applicant_name")
  - application_date = tool_context.invocation_state.get("application_date")
  - いずれかが None → エラー辞書を返して終了
  ↓
[ステップ2] 入力バリデーション
  - ExpenseReportInput(items=items) でバリデーション
  - ValidationError → エラー辞書を返して終了
  ↓
[ステップ3] 共通フォーム生成処理を呼び出す
  - _generate_form(
        template_path=_EXPENSE_TEMPLATE_PATH,
        validated=validated,
        applicant_name=applicant_name,
        application_date=application_date,
        write_detail_rows=_write_expense_detail_rows,
        output_filename="経費精算申請書"
    )
  ↓
[ステップ4] 結果返却
  - _generate_form の戻り値（タプル）を辞書に変換して返す
  - 成功時: (True, file_path) → {"success": True, "file_path": file_path, "error_message": None}
  - 失敗時: (False, msg) → {"success": False, "file_path": None, "error_message": msg}
```

#### 3.3.2 共通フォーム生成処理（_generate_form）

**用途**: `generate_expense_report` と `generate_transport_report` の共通処理を抽出したヘルパー関数

**実装**:
```
_generate_form(template_path, validated, applicant_name, application_date,
               write_detail_rows, output_filename) -> tuple[bool, str]
  ↓
テンプレートファイル存在確認（os.path.exists）
  - 存在しない → (False, "申請書テンプレートが見つかりませんでした。担当者にお問い合わせください。") を返す
  ↓
output/ ディレクトリ作成（os.makedirs(exist_ok=True)）
  ↓
openpyxl でテンプレートを読み込む（load_workbook）
  ↓
ヘッダー情報をセルに書き込む
  - 申請者名セル: applicant_name
  - 申請日セル: application_date
  ↓
write_detail_rows(ws, validated.items) で明細行を書き込む
  ↓
合計金額を計算してセルに書き込む
  ↓
出力ファイルパスを生成
  - f"output/{output_filename}_{datetime.now().strftime('%Y%m%d_%H%M%S')}.xlsx"
  ↓
ファイルを保存（wb.save）
  - IOError → (False, エラーメッセージ) を返す
  - PermissionError → (False, エラーメッセージ) を返す
  - Exception → (False, エラーメッセージ) を返す
  ↓
(True, file_path) を返す
```

### 3.4 エラーハンドリング

#### 3.4.1 エラー処理フロー

```
エラー発生
  ↓
invocation_state 取得失敗
  → エラーメッセージ直接生成
  → _logger.error("invocation_state取得失敗", exc_info=True)
  → {"success": False, "file_path": None, "error_message": msg} を返す

ValidationError（入力バリデーション失敗）
  → ErrorHandler.handle_validation_error(e) でメッセージ生成
  → _logger.error("バリデーションエラー: %s, file_path=None", e, exc_info=True)
  → {"success": False, "file_path": None, "error_message": msg} を返す

Exception（_generate_form 失敗・その他）
  → ErrorHandler.handle_unexpected_error(e) でメッセージ生成
  → _logger.error("経費精算申請書生成エラー: %s, file_path=%s", e, file_path, exc_info=True)
  → {"success": False, "file_path": None, "error_message": msg} を返す
```

#### 3.4.2 例外テーブル（generate_expense_report）

| 例外クラス | 発生条件 | 対応 | 戻り値 |
|-----------|---------|------|--------|
| `ValidationError` | 入力パラメータが不正 | `ErrorHandler.handle_validation_error()` + `_logger.error` (file_path付き) | `{"success": False, "file_path": None, "error_message": msg}` |
| `Exception` | _generate_form 失敗・その他 | `ErrorHandler.handle_unexpected_error()` + `_logger.error` (file_path付き) | `{"success": False, "file_path": None, "error_message": msg}` |

#### 3.4.3 エラーメッセージの詳細

| エラー種別 | 発生条件 | 対応 | ユーザー向けメッセージ |
|-----------|---------|------|---------------------|
| invocation_state 不足（EX-06） | applicant_name / application_date が None | 直接生成 | "申請者情報が取得できませんでした。担当者にお問い合わせください。" |
| バリデーションエラー（EX-01） | 入力パラメータが不正 | ErrorHandler.handle_validation_error() | "入力内容にエラーがあります。{フィールド名}のエラー: {詳細}" |
| テンプレート不存在（EX-06） | テンプレートファイルが存在しない | _generate_form 内で処理 | "申請書テンプレートが見つかりませんでした。担当者にお問い合わせください。" |
| ファイル書き込み失敗（EX-07） | ディスク容量不足・権限不足等 | _generate_form 内で処理 | "申請書の保存に失敗しました。担当者にお問い合わせください。" |
| 想定外エラー（EX-08） | 予期しない Python 例外 | ErrorHandler.handle_unexpected_error() | "予期しないエラーが発生しました。担当者にお問い合わせください。" |

## 6. 補助関数

### 6.1 _generate_form

**目的**: `generate_expense_report` と `generate_transport_report` の共通処理（テンプレート読み込み→セル書き込み→ファイル保存）を抽出したヘルパー関数

**シグネチャ**:
```python
def _generate_form(
    template_path: str,
    validated,
    applicant_name: str,
    application_date: str,
    write_detail_rows,
    output_filename: str,
) -> tuple[bool, str]
```

**処理フロー**:
```
テンプレートファイル存在確認（os.path.exists）
  - 存在しない → (False, "申請書テンプレートが見つかりませんでした。担当者にお問い合わせください。") を返す
  ↓
output/ ディレクトリ作成（os.makedirs(exist_ok=True)）
  ↓
openpyxl.load_workbook(template_path) でテンプレート読み込み
  ↓
ws = wb.active でアクティブシートを取得
  ↓
ヘッダーセルに書き込む
  - 申請者名セル: applicant_name
  - 申請日セル: application_date
  ↓
write_detail_rows(ws, validated.items) で明細行を書き込む
  ↓
合計金額を計算（sum(item.amount for item in validated.items) または sum(item.fare ...)）
合計金額セルに書き込む
  ↓
file_path = f"output/{output_filename}_{datetime.now().strftime('%Y%m%d_%H%M%S')}.xlsx"
wb.save(file_path)
  - IOError → _logger.error("ファイル保存エラー（IO）: %s, file_path=%s", e, file_path, exc_info=True)
             → (False, "申請書の保存に失敗しました。担当者にお問い合わせください。") を返す
  - PermissionError → _logger.error("ファイル保存エラー（権限）: %s, file_path=%s", e, file_path, exc_info=True)
                     → (False, "申請書の保存に失敗しました。ファイルへのアクセス権限がありません。担当者にお問い合わせください。") を返す
  - Exception → _logger.error("ファイル保存エラー（想定外）: %s, file_path=%s", e, file_path, exc_info=True)
              → (False, "予期しないエラーが発生しました。担当者にお問い合わせください。") を返す
  ↓
(True, file_path) を返す
```

**戻り値**: `tuple[bool, str]`
- 成功時: `(True, file_path)`
- 失敗時: `(False, エラーメッセージ文字列)`

**例外テーブル（_generate_form 内 wb.save）**:

| 例外クラス | 発生条件 | 対応 | 戻り値 |
|-----------|---------|------|--------|
| `IOError` | ディスク容量不足・書き込み不可 | `_logger.error` (file_path付き) + メッセージ返却 | `(False, エラーメッセージ)` |
| `PermissionError` | ファイルへのアクセス権限不足 | `_logger.error` (file_path付き) + メッセージ返却 | `(False, エラーメッセージ)` |
| `Exception` | 予期しないエラー | `_logger.error` (file_path付き) + メッセージ返却 | `(False, エラーメッセージ)` |

## 7. 補足情報

### 7.1 実装上の注意点

1. **デコレータ**
   - `invocation_state` を参照するため `@tool(context=True)` を使用する
   - `tool_context: ToolContext` を関数の最後の引数に追加する

2. **出力ファイルパスの自律生成**
   - 出力ファイルパスはツール内部で `datetime.now().strftime("%Y%m%d_%H%M%S")` をもとに生成する
   - LLM のパラメータ（ツール関数の引数）として受け取ってはならない

3. **エラー返却キー名**
   - エラー返却辞書のキー名は `ReportOutput` のフィールド名（`success`, `file_path`, `error_message`）と一致させる

4. **DRY 原則**
   - `generate_expense_report` と `generate_transport_report` の共通処理は `_generate_form` に抽出する（R9.12.1）

5. **承認制御**
   - HumanApprovalHook が `BeforeToolCallEvent` で承認制御を行う。ツール内部では承認チェック不要

6. **ファイル読み込み実装方針**
   - ファイル存在チェックは `os.path.exists()` による事前チェックで実装する
   - `open()` 実行時の `FileNotFoundError` 捕捉との併用は禁止し、事前チェックを優先する
   - テンプレートファイルの存在確認は `_generate_form` 内で `os.path.exists(template_path)` により行い、存在しない場合はタプル `(False, エラーメッセージ)` を即座に返す

### 7.2 パフォーマンス考慮事項

1. **output/ ディレクトリ作成**
   - `os.makedirs(exist_ok=True)` で毎回確認・作成する（初回のみ実際に作成される）

### 7.3 セキュリティ考慮事項

1. **テンプレートパスの固定**
   - テンプレートファイルパスは定数（`_EXPENSE_TEMPLATE_PATH`, `_TRANSPORT_TEMPLATE_PATH`）で管理し、外部から変更不可とする

2. **出力先の固定**
   - 出力先は `output/` ディレクトリに固定し、パストラバーサルを防止する
