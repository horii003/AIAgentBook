## 1. 概要

### 1.1 コンポーネントの目的

収集済み情報（申請者名・移動情報または経費情報・上長承認要否フラグ等）を申請書テンプレートに反映し、Excelファイルを生成する。申請者名・申請日は invocation_state から取得する。

### 1.2 主要な責務

- 申請種別に応じた申請書テンプレート（Excel）の読み込み
- 収集済み情報をテンプレートの所定セルへの書き込み
- 上長承認要否フラグが True の場合の承認欄追加
- 生成した申請書を `output/` ディレクトリへの保存とファイルパス返却
- 入力バリデーション（Pydantic モデル）
- 共通ヘルパー `_generate_form` による DRY 原則の遵守

## 3. generate_transport_expense_form ツール設計

### 3.1 基本情報

#### ツール名（関数名）
`generate_transport_expense_form`

#### 説明（docstring）
```
交通費精算申請書（Excel）を生成する。
申請者名・申請日は invocation_state から取得する。

Args:
    transport_records (list[dict]): 移動区間情報のリスト。各要素は以下のキーを持つ辞書:
        - travel_date (str): 移動日（YYYY-MM-DD形式）【必須】
        - departure (str): 出発地【必須】
        - destination (str): 目的地【必須】
        - transport_type (str): 交通手段（電車・バス・タクシー・飛行機）【必須】
        - fare (int): 運賃（円、0以上）【必須】
        - business_purpose (str): 業務目的【必須】
    needs_approval (bool): 上長承認要否フラグ【必須】

Returns:
    dict: 以下のキーを持つ辞書:
        - success (bool): 生成成否フラグ
        - file_path (str | None): 生成済み申請書ファイルパス。正常時のみ設定。
        - error_message (str | None): エラーメッセージ。エラー時のみ設定。
```

---

### 3.2 インターフェース設計

#### 3.2.1 入力

**パラメータ定義**:

| 引数名 | 型 | 説明 | デフォルト値 |
|--------|-----|------|--------------|
| `transport_records` | `list[dict]` | 移動区間情報のリスト（1件以上必須） | なし（必須） |
| `needs_approval` | `bool` | 上長承認要否フラグ | なし（必須） |
| `tool_context` | `ToolContext` | ツールコンテキスト（invocation_state を含む） | なし（フレームワーク提供） |

**transport_records 各要素のフィールド**:

| フィールド名 | 型 | 必須/任意 | 説明 | 制約 |
|------------|-----|-----------|------|------|
| `travel_date` | `str` | 必須 | 移動日（YYYY-MM-DD形式） | validate_date_format バリデーター |
| `departure` | `str` | 必須 | 出発地 | min_length=1 |
| `destination` | `str` | 必須 | 目的地 | min_length=1 |
| `transport_type` | `str` | 必須 | 交通手段 | normalize_transport_type バリデーター |
| `fare` | `int` | 必須 | 運賃（円） | ge=0 |
| `business_purpose` | `str` | 必須 | 業務目的 | min_length=1 |

##### バリデーションルール詳細

| フィールド | ルール | エラーメッセージ |
|-----------|--------|----------------|
| `transport_records` | 1件以上必須（min_length=1） | "移動情報を1件以上入力してください。" |
| `travel_date` | YYYY-MM-DD形式の有効な日付 | "移動日はYYYY-MM-DD形式で入力してください。" |
| `departure` | 空文字禁止 | "出発地を入力してください。" |
| `destination` | 空文字禁止 | "目的地を入力してください。" |
| `transport_type` | 電車・バス・タクシー・飛行機のいずれか | "交通手段は電車・バス・タクシー・飛行機のいずれかを指定してください。" |
| `fare` | 0以上の整数 | "運賃は0以上の整数で入力してください。" |
| `business_purpose` | 空文字禁止 | "業務目的を入力してください。" |

#### 3.2.2 出力

**戻り値の型**: `dict`

**構造**:
```python
{
    "success": bool,
    "file_path": str | None,
    "error_message": str | None
}
```

#### 3.2.3 例外

| 例外 | 発生条件 |
|------|---------|
| `ValidationError` | 入力パラメータのバリデーション失敗 |
| `IOError` | ファイル書き込み失敗（ディスク容量不足等） |
| `PermissionError` | ファイル書き込み権限エラー |
| `Exception` | その他の予期しないエラー |

### 3.3 ビジネスロジック

#### 3.3.1 処理フロー

```
開始
  ↓
invocation_state から user_name・request_date を取得
  ↓
TransportExpenseFormInput でバリデーション
  - ValidationError → ErrorHandler.handle_validation_error(e)
    → {"success": False, "file_path": None, "error_message": ...} を返す
  ↓
_generate_form(template_path, validated, _write_transport_rows, output_filename) を呼び出す
  ↓
[_generate_form 内]
  os.path.exists(template_path) で存在チェック
  - 存在しない場合 → (False, エラーメッセージ) を返す
    → {"success": False, "file_path": None, "error_message": ...} を返す
  ↓
  openpyxl でテンプレートを読み込む
  ↓
  申請者名・申請日をヘッダーセルに書き込む
  ↓
  _write_transport_rows(ws, validated) で移動区間明細を書き込む
    - A列: No（行番号）
    - B列: 移動日
    - C列: 出発地
    - D列: 目的地
    - E列: 交通手段
    - F列: 費用
    - G列: 業務目的
    - H列: 承認状況（初期値: 空欄）
  ↓
  needs_approval=True の場合、承認欄を追加する
  ↓
  output_filename = f"交通費精算申請書_{datetime.now().strftime('%Y%m%d_%H%M%S')}.xlsx"
  file_path = f"output/{output_filename}"
  os.makedirs("output", exist_ok=True)
  wb.save(file_path) → _save_form(wb, file_path)
    - IOError → (False, エラーメッセージ) を返す
    - PermissionError → (False, エラーメッセージ) を返す
    - Exception → (False, エラーメッセージ) を返す
  ↓
  (True, file_path) を返す
  ↓
{"success": True, "file_path": file_path, "error_message": None} を返す
```

#### 3.3.2 申請書列定義（交通費精算申請書 移動区間明細）

| 列 | 内容 | 初期値 |
|----|------|--------|
| A | No（行番号） | 1, 2, 3... |
| B | 移動日 | travel_date |
| C | 出発地 | departure |
| D | 目的地 | destination |
| E | 交通手段 | transport_type |
| F | 費用 | fare |
| G | 業務目的 | business_purpose |
| H | 承認状況 | 空欄 |

### 3.4 エラーハンドリング

#### 3.4.1 エラー処理フロー

```
[ValidationError]
  → _logger.error("バリデーションエラー（交通費申請書）: ...", exc_info=True)
  → ErrorHandler.handle_validation_error(e)
  → {"success": False, "file_path": None, "error_message": ...} を返す

[テンプレートファイル未発見]
  → _logger.warning("テンプレートファイルが見つかりません: {file_path}")
  → ErrorHandler.handle_file_save_error(FileNotFoundError(...))
  → {"success": False, "file_path": None, "error_message": ...} を返す

[IOError（ファイル保存失敗）]
  → _logger.error("ファイル保存エラー: {file_path}", exc_info=True)
  → ErrorHandler.handle_file_save_error(e)
  → (False, エラーメッセージ) を返す（_save_form 内）

[PermissionError（権限エラー）]
  → _logger.error("ファイル保存権限エラー: {file_path}", exc_info=True)
  → ErrorHandler.handle_file_save_error(e)
  → (False, エラーメッセージ) を返す（_save_form 内）

[Exception（その他）]
  → _logger.error("予期しないエラー（申請書生成）: {file_path}", exc_info=True)
  → ErrorHandler.handle_file_save_error(e)
  → (False, エラーメッセージ) を返す（_save_form 内）
```

#### 3.4.2 エラーメッセージの詳細

| エラー種別 | 発生条件 | 戻り値 | ユーザー向けメッセージ |
|-----------|---------|--------|---------------------|
| ValidationError | 入力パラメータ不正 | `{"success": False, "file_path": None, "error_message": "..."}` | "入力内容に誤りがあります。{詳細}" |
| テンプレート未発見 | テンプレートファイルが存在しない | `{"success": False, "file_path": None, "error_message": "..."}` | "申請書テンプレートが見つかりませんでした。申請管理部門にお問い合わせください。" |
| IOError | ファイル書き込み失敗 | `{"success": False, "file_path": None, "error_message": "..."}` | "申請書ファイルの保存に失敗しました。申請管理部門にお問い合わせください。" |
| PermissionError | 書き込み権限なし | `{"success": False, "file_path": None, "error_message": "..."}` | "申請書ファイルの保存に失敗しました（権限エラー）。申請管理部門にお問い合わせください。" |
| Exception（その他） | 予期しないエラー | `{"success": False, "file_path": None, "error_message": "..."}` | "申請書生成中にエラーが発生しました。申請管理部門にお問い合わせください。" |

## 6. 補助関数

### 6.1 _generate_form

**目的**: テンプレート読み込み・セル書き込み・ファイル保存の共通処理を担う内部ヘルパー関数

**シグネチャ**:
```python
def _generate_form(
    template_path: str,
    validated,
    write_detail_rows,
    output_prefix: str,
    user_name: str,
    request_date: str,
) -> tuple[bool, str]
```

**戻り値**:
- `(True, file_path)`: 生成成功
- `(False, エラーメッセージ文字列)`: 生成失敗

**処理内容**:
1. `os.path.exists(template_path)` で存在チェック。存在しない場合は `(False, エラーメッセージ)` を返す
2. `openpyxl.load_workbook(template_path)` でテンプレートを読み込む
3. ヘッダーセルに申請者名・申請日を書き込む
4. `write_detail_rows(ws, validated)` で明細行を書き込む
5. `needs_approval=True` の場合、承認欄を追加する
6. `output_filename = f"{output_prefix}_{datetime.now().strftime('%Y%m%d_%H%M%S')}.xlsx"`
7. `file_path = f"output/{output_filename}"`
8. `os.makedirs("output", exist_ok=True)`
9. `_save_form(wb, file_path)` でファイルを保存する
10. `(True, file_path)` を返す

### 6.2 _save_form

**目的**: ワークブックをファイルに保存する。IOError・PermissionError・Exception を個別に捕捉する

**シグネチャ**:
```python
def _save_form(wb, file_path: str) -> tuple[bool, str]
```

**戻り値**:
- `(True, "")`: 保存成功
- `(False, エラーメッセージ文字列)`: 保存失敗

**例外処理**:
- `IOError`: `_logger.error(..., exc_info=True)` → `ErrorHandler.handle_file_save_error(e)` → `(False, メッセージ)`
- `PermissionError`: `_logger.error(..., exc_info=True)` → `ErrorHandler.handle_file_save_error(e)` → `(False, メッセージ)`
- `Exception`: `_logger.error(..., exc_info=True)` → `ErrorHandler.handle_file_save_error(e)` → `(False, メッセージ)`

**注意**: `IOError` と `PermissionError` は個別に定義する（`IOError` を `PermissionError` に統合しない）

### 6.3 _write_transport_rows

**目的**: 交通費精算申請書の移動区間明細行を書き込む

## 7. 補足情報

### 7.1 実装上の注意点

1. **申請者名・申請日の取得**
   - `tool_context.invocation_state.get("user_name")` および `tool_context.invocation_state.get("request_date")` で取得する
   - LLM がツールパラメータとして渡す値ではない

2. **ファイル存在チェックの方式**
   - `os.path.exists()` による事前チェックを使用する
   - `open()` 実行時の `FileNotFoundError` 捕捉との併用は禁止する

3. **エラー返却キー名の統一**
   - ツール関数のエラー返却辞書のキー名は `FormGeneratorOutput` のフィールド名（`success`, `file_path`, `error_message`）と一致させる

4. **ファイル保存エラーの例外クラス**
   - `_save_form` では `IOError`・`PermissionError`・`Exception` の3種類を個別に定義する

5. **申請書生成メソッドの例外クラス**
   - `generate_transport_expense_form` / `generate_general_expense_form` では `ValidationError` と `Exception` の2種類のみとする（`FileNotFoundError` は定義しない）

6. **全エラーログに file_path を付加**
   - ファイル操作に関するエラーログには `file_path` を付加情報として含める

7. **HumanApprovalHook との連携**
   - 本ツール呼び出し前に HumanApprovalHook が発火することを前提とした設計とする

---

### 7.2 パフォーマンス考慮事項

1. **出力ディレクトリの自動作成**
   - `os.makedirs("output", exist_ok=True)` で `output/` ディレクトリを自動作成する

---

### 7.3 セキュリティ考慮事項

1. **入力バリデーション**
   - 全入力パラメータを Pydantic モデルでバリデーションし、不正な値を拒否する
