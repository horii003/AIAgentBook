## 1. 概要

### 1.1 コンポーネントの目的

収集した申請情報をExcelテンプレートに書き込み、申請書ファイルを `output/` ディレクトリに保存する。生成した申請書のファイルパスをエージェントに返し、ユーザーへ通知させる。HumanApprovalHookによりユーザーのOK選択後のみ実行される（APR-001）。

### 1.2 主要な責務

- **テンプレート読み込み**: 申請種別に対応するExcelテンプレートファイルを読み込む
- **申請情報書き込み**: 収集済み申請情報をテンプレートの所定セルに書き込む
- **ファイルパス自律生成**: 出力ファイルパスはツール内部で `datetime.now().strftime("%Y%m%d_%H%M%S")` をもとに生成する（LLMのパラメータとして受け取らない）
- **ファイル保存**: タイムスタンプ付きファイル名で `output/` ディレクトリに保存する
- **結果返却**: 生成結果（成功/失敗）とファイルパスをエージェントに返す
- **invocation_state参照**: 申請者名・申請日は `invocation_state` から取得する

## 3. generate_transport_form ツール設計

### 3.1 基本情報

#### ツール名（関数名）
`generate_transport_form`

#### 説明（docstring）
```
交通費精算申請書（Excel）を生成してoutput/ディレクトリに保存する。

HumanApprovalHookによりユーザーのOK選択後のみ実行される。
申請者名・申請日はinvocation_stateから取得する。
出力ファイルパスはツール内部でタイムスタンプをもとに自律生成する。

Args:
    items (list[dict]): 移動明細リスト（min_length=1）。各要素は以下のキーを持つ辞書:
        - travel_date (str): 移動日（YYYY-MM-DD形式）【必須】
        - departure (str): 出発地【必須】
        - destination (str): 目的地【必須】
        - transport_type (str): 交通手段（電車・バス・タクシー・飛行機）【必須】
        - fare (float): 交通費（円、0以上）【必須】
        - purpose (str): 業務目的（空文字禁止）【必須】

Returns:
    dict: 以下のキーを持つ辞書:
        - success (bool): 処理成功フラグ
        - file_path (str): 生成ファイルパス。success=True の場合のみ有効
        - error_message (str): エラーメッセージ。success=False の場合のみ有効
```

### 3.2 インターフェース設計

#### 3.2.1 入力

**パラメータ定義**:

| 引数名 | 型 | 説明 | デフォルト値 |
|--------|-----|------|--------------|
| `items` | `list[dict]` | 移動明細リスト | なし（必須） |
| `tool_context` | `ToolContext` | ツールコンテキスト（invocation_stateを含む） | なし（フレームワーク注入） |

**items 各要素のフィールド**:

| フィールド名 | 型 | 必須/任意 | 説明 | 制約 |
|------------|-----|-----------|------|------|
| `travel_date` | `str` | 必須 | 移動日 | YYYY-MM-DD形式 |
| `departure` | `str` | 必須 | 出発地 | min_length=1 |
| `destination` | `str` | 必須 | 目的地 | min_length=1 |
| `transport_type` | `str` | 必須 | 交通手段 | 電車・バス・タクシー・飛行機のいずれか |
| `fare` | `float` | 必須 | 交通費（円） | ge=0 |
| `purpose` | `str` | 必須 | 業務目的 | min_length=1（BRL-09） |

**invocation_state から取得するフィールド**:

| フィールド名 | 取得方法 | 説明 |
|------------|---------|------|
| `applicant_name` | `tool_context.invocation_state.get("applicant_name")` | 申請者名 |
| `application_date` | `tool_context.invocation_state.get("application_date")` | 申請日（YYYY-MM-DD形式） |

##### バリデーションルール詳細

| フィールド | ルール | エラーメッセージ |
|-----------|--------|----------------|
| `items` | min_length=1（空リスト禁止） | "移動明細のエラー: 移動明細が入力されていません。対処方法: 移動明細を入力してください。" |
| `applicant_name` | min_length=1（空文字禁止） | "申請者名のエラー: 申請者名が取得できませんでした。" |
| `application_date` | YYYY-MM-DD形式 | "申請日のエラー: 日付の形式が正しくありません。" |
| `items[].travel_date` | YYYY-MM-DD形式 | "移動日のエラー: 日付の形式が正しくありません。" |
| `items[].fare` | ge=0 | "交通費のエラー: 交通費は0以上の値を入力してください。" |
| `items[].purpose` | min_length=1 | "業務目的のエラー: 業務目的が入力されていません。" |

#### 3.2.2 出力

**戻り値の型**: `dict`

**構造**:
```python
{
    "success": bool,        # 処理成功フラグ
    "file_path": str,       # 生成ファイルパス（例: "output/交通費精算申請書_20260510_143022.xlsx"）
    "error_message": str,   # エラーメッセージ（success=False の場合のみ有効）
}
```

### 3.3 ビジネスロジック

#### 3.3.1 処理フロー

```
開始
  ↓
[ステップ1] invocation_state から申請者名・申請日を取得
  - applicant_name = tool_context.invocation_state.get("applicant_name")
  - application_date = tool_context.invocation_state.get("application_date")
  ↓
[ステップ2] 入力バリデーション
  - TransportFormInput(applicant_name, application_date, items) でバリデーション
  - ValidationError → エラーメッセージを返す（EX-01）
  ↓
[ステップ3] テンプレートファイル存在確認
  - os.path.exists(_TRANSPORT_TEMPLATE_PATH) が False → エラーメッセージを返す（EX-06）
  ↓
[ステップ4] 出力ファイルパス生成
  - timestamp = datetime.now().strftime("%Y%m%d_%H%M%S")
  - file_path = f"output/交通費精算申請書_{timestamp}.xlsx"
  - os.makedirs("output", exist_ok=True)
  ↓
[ステップ5] Excelテンプレート読み込み・書き込み
  - openpyxl.load_workbook(_TRANSPORT_TEMPLATE_PATH) でテンプレートを読み込む
  - 申請者名・申請日をヘッダーセルに書き込む
  - 移動明細リストを明細行に書き込む（_write_transport_rows）
  ↓
[ステップ6] ファイル保存
  - workbook.save(file_path)
  ↓
[ステップ7] 結果を返す
  - {"success": True, "file_path": file_path, "error_message": ""}
```

#### 3.3.2 Excel列定義（交通費精算申請書 移動区間明細）

| 列 | フィールド | 説明 |
|----|-----------|------|
| A | No（行番号） | 1から始まる連番 |
| B | 移動日 | `travel_date`（YYYY-MM-DD） |
| C | 出発地 | `departure` |
| D | 目的地 | `destination` |
| E | 交通手段 | `transport_type` |
| F | 費用 | `fare`（円） |
| G | 業務目的 | `purpose` |
| H | 承認状況 | 初期値: 空欄 |

#### 3.3.3 データの検証方法

**Pydanticモデル**: `TransportFormInput`

**検証内容**:
- `applicant_name`: min_length=1
- `application_date`: validate_date_format で YYYY-MM-DD 形式を検証
- `items`: min_length=1（空リスト禁止）
- `items[]`: `TransportItem` モデルで各フィールドをバリデーション

### 3.4 エラーハンドリング

#### 3.4.1 エラー処理フロー

```
エラー発生
  ↓
[ValidationError（EX-01）: generate_transport_form 内]
  → ErrorHandler.handle_validation_error(e) に委譲
  → {"success": False, "file_path": "", "error_message": message} を返す
  ↓
[IOError（EX-06）: ファイル保存時]
  → ErrorHandler.handle_file_save_error(e) に委譲
  → (False, エラーメッセージ文字列) を返す（tuple[bool, str]）
  ↓
[PermissionError（EX-06）: ファイル保存時]
  → ErrorHandler.handle_file_save_error(e) に委譲
  → (False, エラーメッセージ文字列) を返す（tuple[bool, str]）
  ↓
[Exception（EX-08）: ファイル保存時]
  → ErrorHandler.handle_file_save_error(e) に委譲
  → (False, エラーメッセージ文字列) を返す（tuple[bool, str]）
  ↓
[Exception（EX-08）: generate_transport_form 内]
  → ErrorHandler.handle_unexpected_error(e) に委譲
  → {"success": False, "file_path": "", "error_message": message} を返す
```

#### 3.4.2 エラーメッセージの詳細

| エラー種別 | 発生条件 | 発生箇所 | 対応 | 戻り値形式 | ユーザー向けメッセージ |
|-----------|---------|---------|------|-----------|---------------------|
| ValidationError（EX-01） | 必須項目未入力・不正形式 | generate_transport_form | ErrorHandler.handle_validation_error() | dict | "移動明細のエラー: ..." 等 |
| IOError（EX-06） | ファイル書き込みエラー | _save_workbook | ErrorHandler.handle_file_save_error() | tuple[bool, str] | "申し訳ありません。申請書の保存に失敗しました。システム管理者にご連絡ください。" |
| PermissionError（EX-06） | ファイル書き込み権限なし | _save_workbook | ErrorHandler.handle_file_save_error() | tuple[bool, str] | "申し訳ありません。申請書の保存に失敗しました。ファイルへのアクセス権限を確認してください。" |
| Exception（EX-08） | 予期しない保存エラー | _save_workbook | ErrorHandler.handle_file_save_error() | tuple[bool, str] | "申し訳ありません。申請書の保存中に予期しないエラーが発生しました。" |
| Exception（EX-08） | 予期しない例外 | generate_transport_form | ErrorHandler.handle_unexpected_error() | dict | "申し訳ありません。予期しないエラーが発生しました。システム管理者にご連絡ください。" |

| ERROR | ファイル書き込みエラー時（Exception） | `"申請書の保存中に予期しないエラーが発生しました: {file_path}"` （exc_info=True） |
| ERROR | 予期しないエラー時 | `"申請書生成中に予期しないエラーが発生しました"` （exc_info=True） |

## 5. 補助関数設計

### 5.1 _write_transport_rows

**目的**: 交通費精算申請書の移動明細行をExcelシートに書き込む

**シグネチャ**: `def _write_transport_rows(ws, items: list[TransportItem], start_row: int) -> None`

**処理**: `items` の各要素を `start_row` から順に1行ずつ書き込む（列定義: A=No, B=移動日, C=出発地, D=目的地, E=交通手段, F=費用, G=業務目的, H=承認状況（空欄））

### 5.2 _write_expense_rows

**目的**: 経費精算申請書の経費明細行をExcelシートに書き込む

**シグネチャ**: `def _write_expense_rows(ws, items: list[ExpenseItem], start_row: int) -> None`

**処理**: `items` の各要素を `start_row` から順に1行ずつ書き込む（列定義: A=No, B=購入日, C=店舗名, D=品目, E=経費区分, F=金額, G=業務目的, H=承認状況（空欄））

### 5.3 _save_workbook

**目的**: Excelワークブックをファイルに保存する

**シグネチャ**: `def _save_workbook(workbook, file_path: str) -> tuple[bool, str]`

**処理フロー**:
```
開始
  ↓
workbook.save(file_path)
  ↓
(True, "") を返す
  ↓
[IOError 発生時]
  → _logger.error("申請書の保存に失敗しました（IOError）: {file_path}", exc_info=True)
  → (False, ErrorHandler.handle_file_save_error(e)) を返す
  ↓
[PermissionError 発生時]
  → _logger.error("申請書の保存に失敗しました（PermissionError）: {file_path}", exc_info=True)
  → (False, ErrorHandler.handle_file_save_error(e)) を返す
  ↓
[Exception 発生時]
  → _logger.error("申請書の保存中に予期しないエラーが発生しました: {file_path}", exc_info=True)
  → (False, ErrorHandler.handle_file_save_error(e)) を返す
```

**戻り値**: `tuple[bool, str]` — (成功フラグ, エラーメッセージ)

## 7. 補足情報

### 7.1 実装上の注意点

1. **デコレータ**
   - `@tool(context=True)` を使用する（invocation_stateから申請者名・申請日を取得するため）
   - 関数の最後の引数に `tool_context: ToolContext` を追加する

2. **ファイルパスの自律生成**
   - 出力ファイルパスはツール内部で生成する（LLMのパラメータとして受け取らない）
   - `datetime.now().strftime("%Y%m%d_%H%M%S")` を使用する

3. **output/ ディレクトリの自動作成**
   - `os.makedirs("output", exist_ok=True)` で存在しない場合は自動作成する

4. **エラー返却キー名**
   - 戻り値辞書のキー名は `TransportFormOutput` / `ExpenseFormOutput` のフィールド名と一致させること
   - `error_message` キーを使用する（`message` は不可）

5. **HumanApprovalHook との連携**
   - `generate_transport_form` と `generate_expense_form` を HumanApprovalHook の `target_tools` に登録すること
   - HumanApprovalHook の承認対象ツール名は本設計書に定義された全ツール関数名（`generate_transport_form`・`generate_expense_form`）と完全一致させること

6. **ファイル保存のエラー処理**
   - ファイル保存は `_save_workbook()` ヘルパーに委譲し、`IOError`・`PermissionError`・`Exception` を個別に捕捉する
   - 各エラーのログには `file_path` を付加情報として含める
   - `_save_workbook()` の戻り値は `tuple[bool, str]` 形式とする

7. **generate() のエラー処理**
   - `generate_transport_form()` / `generate_expense_form()` 内では `ValidationError` と `Exception` の2種類のみ捕捉する
   - `FileNotFoundError` は個別に捕捉しない（テンプレート不存在は `os.path.exists()` 事前チェックで対応）

### 7.2 パフォーマンス考慮事項

1. **冪等性の確保**
   - タイムスタンプ付きファイル名により、再実行時も別ファイルとして生成される

### 7.3 セキュリティ考慮事項

1. **テンプレートパスの固定**
   - テンプレートファイルパスはモジュール定数で管理する
   - LLMからのパラメータとしてファイルパスを受け取らない
