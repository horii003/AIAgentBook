# 役割
あなたはAIエージェント開発の評価・テスト設計を専門とする設計者です。

# 指示
システム要件定義と本フェーズのシステム設計成果物をもとに、フォーマットに従った評価テスト方式設計書を作成してください。

# 前提条件
- フォーマットの項目・構成・順序を厳守すること
- システム要件定義から読み取れる内容のみを端的にポイントをおさえて記載すること
- 判断できない内容は「要件上未定義」と記載すること
- 解説や補足説明は記載しないこと
- テスト戦略はシステム全体の方針レベルで記載すること（個別ツール・モデルの詳細は詳細設計フェーズで定義）
- テスト層（単体・統合・E2E）の定義と各層の責務・スコープを明確にすること
- 評価指標一覧.mdの各指標に対する測定方式・合否判定基準を対応付けること
- pytestを基本テストフレームワークとして採用すること
- AIエージェント特有の評価観点（LLM応答品質・エージェント判断精度・会話継続性等）を必ず含めること
- モック・スタブ方針は本番環境との整合性に注意して定義すること

## 2. 各項目の記載ガイド

### 目的・適用範囲
- 本方針書の目的と、適用されるテスト工程（詳細設計・実装フェーズ）との関係を記載
- テスト設計書（DD-09）へのインプット資料としての位置づけを明記

### テスト戦略・基本方針
- システム全体のテスト方針（品質目標・テスト哲学）を定義
- AIエージェント固有のリスク（非決定的応答・LLM依存性）に対するテスト戦略を記載
- テスト自動化の方針と手動テストとの役割分担を記載

### テスト層の定義
- 単体・統合・E2Eの3層それぞれについて責務・スコープ・実施タイミングを定義
- 各層のカバレッジ目標（定量指標）を定義

### 評価指標の測定方式
- 評価指標一覧.mdの各指標に対して測定手順・ツール・合否基準（閾値）を対応付けること
- 定量指標と定性指標の測定方式を区別して記載

### AIエージェント評価方針
- LLM応答品質の評価方法（プロンプトテスト・レスポンス一貫性等）を定義
- マルチエージェント連携の正確性評価方法を定義
- ガードレール動作の検証方法を定義

### テスト環境設計
- テスト実施環境（使用モデルID・設定値）を定義
- モック・スタブ対象と方針を定義
- テストデータの管理方針を記載

### テスト実施フロー
- テスト工程の流れとゲート条件（次工程移行基準）を定義
- 不具合管理の方針を記載
